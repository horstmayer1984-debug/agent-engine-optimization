# Lista de verificación de AEO SEO: 25 comprobaciones para la visibilidad de búsqueda lista para agentes

Una lista de verificación de AEO SEO debe cubrir tres capas: SEO clásico, legibilidad de respuestas de IA y ejecución lista para el agente. Comience con páginas rastreables y contenido útil, luego agregue respuestas extraíbles, datos estructurados, enlaces internos, llms.txt, políticas y rutas de acción que los agentes autónomos puedan entender.

## Capa 1: base de SEO

La documentación de SEO de Google todavía comienza con lo básico: ayudar a los motores de búsqueda a rastrear, indexar, comprender y mostrar su contenido. No te saltes esa capa.

| Consultar | Condición de aprobación |
|
---|
---|
| 1. Rastreabilidad | Las URL importantes devuelven 200 y no están bloqueadas |
| 2. Indexabilidad | Las páginas importantes no llevan ningún índice |
| 3. Canónicos | Cada página tiene una URL canónica correcta |
| 4. Títulos | Los títulos son únicos, descriptivos y dignos de hacer clic |
| 5. Meta descripciones | Las descripciones explican el valor de la página |
| 6. H1 | Un H1 claro coincide con la intención de la página |
| 7. Enlaces internos | Las páginas importantes están vinculadas desde centros relevantes |
| 8. Mapa del sitio | El mapa del sitio XML incluye URL canónicas indexables |

Para obtener un flujo de auditoría completo, utilice la [auditoría de preparación de AEO](/es/docs/audit/).

## Capa 2: preparación para la respuesta de la IA

Los sistemas de respuesta de IA necesitan una estructura extraíble. Eso no significa escritura robótica. Significa respuestas claras, buenos títulos y afirmaciones respaldadas por las fuentes.

| Consultar | Condición de aprobación |
|
---|
---|
| 9. Respuesta directa | Las primeras 80 palabras responden a la consulta principal |
| 10. Preguntas H2 | Las secciones coinciden con preguntas de seguimiento reales |
| 11. Mesas | Se estructuran comparaciones y criterios |
| 12. Definiciones | Los términos importantes se definen una vez y de forma coherente |
| 13. Fuentes | Enlace de reclamaciones a fuentes primarias o confiables |
| 14. Preguntas frecuentes | Las preguntas frecuentes responden a preguntas de búsqueda reales |
| 15. Coherencia de la entidad | Nombres, protocolos, productos y conceptos son estables |
| 16. Frescura del contenido | Las páginas muestran el protocolo actual o el estado del mercado |

El [marco de optimización de búsqueda de IA](/es/docs/ai-search-optimization/) explica esta capa con más detalle.

## Capa 3: Optimización del motor del agente

Agent Engine Optimization comienza cuando el sistema se puede utilizar, no solo leer.

| Consultar | Condición de aprobación |
|
---|
---|
| 17. llms.txt | Se enumeran páginas importantes para el descubrimiento de agentes |
| 18. Políticas legibles por máquina | Las devoluciones, los precios, la elegibilidad y los límites son claros |
| 19. Datos de productos o servicios | Los agentes pueden comparar atributos sin adivinar |
| 20. Ruta de acción | Los formularios, carritos, API o pasos de reserva son estables |
| 21. Modelo de autorización y permiso | Las acciones de alto riesgo requieren aprobación o autoridad específica |
| 22. Estados de error | Las fallas devuelven estados útiles y próximos pasos |
| 23. Estados de confirmación | Las acciones realizadas son verificables |
| 24. Atribución | Se miden las referencias de IA y las sesiones de origen del agente |
| 25. Bucle de retroalimentación | Las fallas de los agentes se convierten en soluciones |

La [capa de ejecución](/es/docs/execution-layer/) y la [guía de implementación AEO](/es/docs/implement-aeo/) cubren esta capa.

## Cómo utilizar la lista de verificación

Comience con las páginas que ya reciben impresiones de `AEO SEO`, `Agent Engine Optimization` o términos relacionados. El informe de rendimiento de Search Console puede mostrar qué consultas y páginas están ganando visibilidad. Luego, mejore las páginas con mayor impresión antes de crear páginas nuevas.

Este orden importa:

1. Arreglar la página de clasificación.
2. Agregue una o dos páginas de soporte para distintos subtemas.
3. Vincúlelos.
4. Mida el movimiento de consultas y el CTR.
5. Expanda solo cuando aparezca una nueva intención de búsqueda.

## Preguntas frecuentes

### ¿Esta lista de verificación es para SEO o AEO?

Ambos. Comienza con SEO porque AEO necesita descubrimiento, luego agrega verificaciones de capa de ejecución y respuesta de IA.

### ¿Cada página debería pasar las 25 comprobaciones?

No. Es posible que las páginas informativas no necesiten puntos finales de acción. Las páginas transaccionales y de comercio electrónico necesitan más comprobaciones en la capa de ejecución.

### ¿Cuál es la victoria más rápida?

Mejore las páginas que ya tienen impresiones pero un CTR débil: título, meta descripción, respuesta directa, enlaces internos y estructura de preguntas frecuentes.

### ¿Con qué frecuencia se debe repetir la lista de verificación?

Repítalo después de cambios importantes de contenido, actualizaciones de protocolo, lanzamientos de productos y cambios de consultas en Search Console.
