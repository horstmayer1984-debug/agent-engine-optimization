# Lista de verificação de SEO AEO: 25 verificações de visibilidade de pesquisa pronta para agente

Uma lista de verificação de SEO AEO deve cobrir três camadas: SEO clássico, legibilidade de resposta de IA e execução pronta para agente. Comece com páginas rastreáveis ​​e conteúdo útil e, em seguida, adicione respostas pullable, dados estruturados, links internos, llms.txt, políticas e caminhos de ação que os agentes autônomos possam entender.

## Camada 1: base de SEO

A documentação de SEO do Google ainda começa com o básico: ajudar os mecanismos de pesquisa a rastrear, indexar, compreender e exibir seu conteúdo. Não pule essa camada.

| Consulte | Condição de aprovação |
|
---|
---|
| 1. Rastreabilidade | URLs importantes retornam 200 e não são bloqueadas |
| 2. Indexabilidade | Páginas importantes não possuem índice |
| 3. Canônicos | Cada página possui um URL canônico correto |
| 4. Títulos | Os títulos são únicos, descritivos e dignos de cliques |
| 5. Meta descrições | As descrições explicam o valor da página |
| 6.H1 | Um H1 claro corresponde à intenção da página |
| 7. Links internos | Páginas importantes têm links de hubs relevantes |
| 8. Mapa do site | Sitemap XML inclui URLs canônicos indexáveis ​​|

Para um fluxo de auditoria completo, use [Auditoria de Preparação AEO](/es/docs/audit/).

## Camada 2: Prepare-se para a resposta da IA

Os sistemas de resposta de IA precisam de uma estrutura removível. Isso não significa escrita robótica. Significa respostas claras, bons títulos e declarações apoiadas por fontes.

| Consulte | Condição de aprovação |
|
---|
---|
| 9. Resposta direta | As primeiras 80 palavras respondem à consulta principal |
| 10. Questões H2 | As seções correspondem a perguntas reais de acompanhamento |
| 11. Tabelas | Comparações e critérios são estruturados |
| 12. Definições | Termos importantes são definidos de uma só vez e de forma consistente |
| 13. Fontes | Vincular reclamações a fontes primárias ou confiáveis ​​|
| 14. Perguntas frequentes | FAQ responde a perguntas reais de pesquisa |
| 15. Coerência da entidade | Nomes, protocolos, produtos e conceitos são estáveis ​​|
| 16. Atualização de conteúdo | As páginas mostram o protocolo atual ou status do mercado |

A [estrutura de otimização de pesquisa de IA] (/es/docs/ai-search-optimization/) explica essa camada com mais detalhes.

A Otimização do Agent Engine começa quando o sistema pode ser usado, não apenas lido.

## Como usar a lista de verificaçãoComece com páginas que já recebem impressões de `AEO SEO`, `Agent Engine Optimization` ou termos relacionados. O relatório de desempenho do Search Console pode mostrar quais consultas e páginas estão ganhando visibilidade. Em seguida, aprimore as páginas com mais impressão antes de criar novas páginas.

Esta ordem é importante:

1. Corrija a classificação da página.
2. Adicione uma ou duas páginas de suporte para diferentes subtópicos.
3. Vincule-os.
4. Avalie o movimento da consulta e a CTR.
5. Expanda somente quando uma nova intenção de pesquisa aparecer.

## Perguntas frequentes

### Esta lista de verificação é para SEO ou AEO?

Ambos. Tudo começa com SEO porque o AEO precisa de descoberta e, em seguida, adiciona verificações da camada de execução e resposta de IA.

### Cada página deve passar em todas as 25 verificações?

Não. As páginas informativas podem não precisar de pontos de extremidade de ação. As páginas transacionais e de comércio eletrônico precisam de mais verificações na camada de tempo de execução.

### Qual é a vitória mais rápida?

Melhore as páginas que já possuem impressões, mas CTR fraca: título, meta descrição, resposta direta, links internos e estrutura de FAQ.

### Com que frequência a lista de verificação deve ser repetida?

Repita após grandes alterações de conteúdo, atualizações de protocolo, lançamentos de produtos e alterações de consulta no Search Console.
