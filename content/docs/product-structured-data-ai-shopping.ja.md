# AI 購入エージェント向けの構造化された商品データ

AI 購買エージェントには、名前、価格、在庫状況、バリエーション、配送、返品、評価、識別子など、信頼できる製品データが必要です。 Googleは、構造化された商品データは検索がその詳細を理解するのに役立つと述べ、OpenAIは、販売業者はChatGPT商品発見のために明確な構造化された商品情報を公開する必要があると述べている。 e コマース チームにとって、最優先事項はスマート コピーではありません。製品記録を機械読み取り可能かつ最新のものにしています。

## Google の 2 つの主要な製品マーキング ルート

Google では、商品スニペットと販売者リストを次のように区別しています。

|マーキングタイプ |に最適 |
|
---|
---|
|製品のスニペット |購入なしのエディトリアルページまたは製品ページ |
|販売者リスト |バイヤーが製品を直接購入できるページ |

- [Google 構造化商品データ](https://developers.google.com/search/docs/appearance/structured-data/product)
- [Google 販売者リストの構造化データ](https://developers.google.com/search/docs/appearance/structured-data/merchant-listing)
- [OpenAI: ChatGPT による製品の発見を支援](https://openai.com/chatgpt/search-product-discovery/)

## これが AI ショッピングにとって重要な理由

エージェントはスローガンではなく制限を比較します。購入者は次のように尋ねるかもしれません。

- 上限価格未満
- 特定のサイズで利用可能
- 1台のデバイスと互換性があります
- 日付までに納品可能
- 返品対象
- しきい値を超えて分類される

そのデータが欠落していたり、隠されていたり、古い場合は、自信を持って製品を選択することがさらに困難になります。

[ChatGPT 製品推奨チェックリスト] (/docs/chatgpt-product-recommendations-seo/) では、会話による発見について説明しています。このガイドでは、以下の構造化データベースに焦点を当てます。

## 好調な主な分野

|フィールド |エージェントが気にする理由 |
|
---|
---|
| `名前` |実際の商品を特定する |
| `画像` |視覚的なマッチングとより豊富な結果をサポート |
| `オファー.価格` |価格を比較できます |
| `オファー.可用性` |古い推奨事項を防止 |
| `ブランド` |エンティティ マッチングのヘルプ |
|バリエーション |色、サイズ、モデルの誤った選択を避ける |
|発送詳細 |配送制限への対応 |
|返品ポリシー |購入の不確実性を軽減 |
| GTIN または識別子 |カタログマッチングの改善 |

Google では、価格や在庫状況など、すぐに変化する値については、最初の HTML に「Product」構造化データを配置することを推奨しています。

## 構造化データとフィードの比較

|データソース |強さ |
|
---|
---|
|ページ上の構造化データ |正規の製品 URL に直接リンクされています |
|マーチャント センター ニュース |スケールカタログレベルの更新 |
|表示可能な HTML |人間とエージェントに判読可能な確認情報を提供します。

可能であれば両方を使用してください。 Google は、Web ページからの構造化データと販売者のフィードが連携できることを明確に指摘しています。

## 導入チェックリスト

1. ページに適切なマークアップ クラスを使用します。2.価格と在庫状況を実際のオファーと同期させます。
3. 製品が大幅に変更される場合は、バリアント データを含めます。
4. 対象となる場合は、配送と返品の詳細を追加します。
5. 表示されるページのコンテンツを構造化データと一致させます。
6. Search Console とリッチリザルト ツールを使用して検証します。
7. テンプレートを変更した後、再テストします。[e コマース AEO ガイド] (/docs/aeo-ecommerce/) では、商品データが広範なエージェントの準備にどのように適合するかを示しています。 [AI クローラー ガイド](/es/docs/ai-crawlers-robots-txt/) では、ページがアクセス可能でなければならない理由について説明しています。

##構造化データでは修正できないもの

構造化データは次の場合には修正されません。

- 薄い商品説明
- 誤解を招く価格
- 弱いイメージ
- 壊れた支払い
- ポリシーの詳細がありません
- 不可能な配達の約束

エージェントがオファーを理解できるようにします。悪い取引が良い取引になるわけではありません。

## よくある質問

### 製品概要は AI による購買の可視化に十分ですか?

いいえ。これはマシンの理解に役立ちますが、エージェントにはクロール アクセス、正確に表示可能なコンテンツ、信頼性の高いビジネス オペレーションも必要です。

### すべての e コマース ページで販売者リスト タグを使用する必要がありますか?

顧客が商品を直接購入できるページのみが、マーチャントのリスティング エクスペリエンスの対象となります。

### 商品フィードはページ上のスキーマを置き換えますか?

いいえ、Google は両方を併用すると便利だと述べています。

### まず何を修正する必要がありますか?

在庫状況、価格、識別子、バリエーションの明確さ。そこで間違いを犯すと、より早く信頼が失われます。

## 結論

構造化された製品データは、製品の確実性を表す言語です。 AI 購入エージェントにとって、検討されるか無視されるかの違いは、多くの場合確実性です。
