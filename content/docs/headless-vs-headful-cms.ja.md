# ヘッドレス CMS とヘッド付き CMS: 意味、SEO、ハイブリッド オプション

強力な CMS は、コンテンツとプレゼンテーションの両方を制御します。ヘッドレス CMS はコンテンツをプレゼンテーションから分離し、API 経由でコンテンツを配信します。ハイブリッド CMS は両方のパターンのサポートを試みます。 SEO にとって最良のオプションは、チームがクロール可能で適切に構造化された内部リンクされたページを一貫して公開するために使用できるオプションです。

## 見出し付き CMS とは何を意味しますか?

ヘッドフル CMS は、従来の CMS または結合された CMS の別の用語です。 「ヘッダー」はインターフェイスです。テンプレート、テーマ、ページ ビルダー、またはプレゼンテーション レイヤーです。

強力な CMS では、編集者は通常次のことを行うことができます。

- ページを作成する
- レイアウトを編集する
- コンテンツのプレビュー
- ナビゲーションを管理する
- 個別のインターフェース実装なしで公開する

これにより、ヘッドフル システムが多くのチームにとって実用的になります。

## ヘッドレス CMS とは何ですか?

ヘッドレス CMS では、固定プレゼンテーション層が不要になります。コンテンツを保存し、API 経由でインターフェイスに送信します。開発者はインターフェースを個別に作成します。

これは、Web サイト、アプリ、ドキュメント ポータル、キオスク、電子メール、AI 読み取り可能なサーフェスなど、コンテンツを複数の場所に表示する必要がある場合に役立ちます。

より広い概念については、[ヘッドレス ソフトウェアとは?](/es/docs/what-is-headless-software/) を読んでください。

## ハイブリッド CMS とは何ですか

ハイブリッド CMS は、ヘッドレス配信とヘッドレス配信の両方をサポートします。これにより、パブリッシャーは API 経由でコンテンツを公開しながら、CMS 内でページを作成できるようになります。

ハイブリッドは次のような場合に役立ちます。

- 編集者は視覚的なコントロールが必要です
- 開発者はAPIアクセスが必要です
- ウェブサイトはメインのままです
- コンテンツはアプリケーションまたは人工知能システムにもフィードします
- 移行は徐々に行わなければなりません

Adobe のヘッドレス ドキュメントでは、より広範な CMS 環境内のコンテンツ モデル、コンテンツ フラグメント、GraphQL API などのヘッドレス機能について説明します。

## SEO への影響

Headful CMS SEO は通常、コンテンツの品質、プラグインの過負荷、ページの重複、またはテーマの貧弱さが原因で失敗します。

ヘッドレス CMS SEO は通常、実装が欠落しているために失敗します。

- レンダリングされていないメタデータ
- JavaScript の背後に隠されたコンテンツ
- インデックスされたページのプレビュー
- 追跡不可能な内部リンク
- サイトマップが生成されない
- 不完全な画像フィールド

サイトが分離されている場合は、[ヘッドレス SEO](/es/docs/headless-seo/) と [ヘッドレス CMS のイメージ SEO](/es/docs/image-seo-headless-cms/) を使用してください。## AEO への影響

[エージェント エンジンの最適化](/es/docs/what-is-aeo/) の場合、ハイブリッドおよびヘッドレス システムにより、コンテンツを構造化データとして公開しやすくなります。しかし、ヘッドフル システムは、しっかりとした公開ページを作成する場合には依然として機能します。

鍵はラベルではありません。エージェントが発見し、理解し、引用し、最終的に行動できるかどうかが重要です。

## よくある質問

### headful CMS は従来の CMS と同じですか?

一般的にはそうです。ヘッドフルとは、CMS にプレゼンテーション層が含まれるか、プレゼンテーション層が制御されることを意味します。### ヘッドレス CMS はヘッド付き CMS よりも優れていますか?

プロジェクトがフロントエンドの柔軟性、マルチチャネル公開、構造化コンテンツの再利用、または API アクセスを必要とする場合のみ。

### ハイブリッド CMS とは何ですか?

ハイブリッド CMS は、従来のページ サービスと API ベースのヘッドレス配信をサポートします。

### どのモデルが SEO に最適ですか?

正しいメタデータと有用なコンテンツを備えた、高速でクロール可能、ユニークでリンクの充実したページを生成するモデル。
