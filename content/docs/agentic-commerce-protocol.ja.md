#エージェント取引プロトコル (ACP): トレーダーが知っておくべきこと

Agentic Commerce Protocol は、AI エージェントでのコンテキスト内販売を可能にする Stripe ドキュメント プロトコル層です。実際的に言えば、ACP はエージェントのエクスペリエンス内で支払いを行うことを目的としています。つまり、ユーザーは AI サーフェス上で商品を発見し、購入フローをコンテキストに保ち、販売者は人間のブラウザ セッションを中心に店頭全体を再構築することなく、そのフローに接続します。

## 公式文書に記載されていること

Stripe Agent Commerce のドキュメントでは、Agent Commerce プロトコルを使用して AI エージェント上でインコンテキスト販売を可能にするよう企業に指示されています。 Stripe の発表発表では、ACP が OpenAI と共同開発されたオープンスタンダードであるとも説明されています。これにより、ACP は、完全な検索、製品データ、またはマルチエージェント オーケストレーション スタックではなく、トランザクション エクスペリエンス レイヤーに直接配置されます。

## ACP と近隣プロトコルの比較

|プロトコル |主な仕事 |
|
---|
---|
| ACP | AI エージェント エクスペリエンスのコンテキスト内支払い |
| UCP |広範なビジネス ライフサイクルと機能の交渉 |
| PCM |エージェント向けのツールとデータへのアクセス |
| x402 | HTTP リソースの自動支払いフロー |

[UCP、ACP、および MCP ガイド](/es/docs/ucp-vs-acp-vs-mcp/) では、より広範な比較がカバーされています。 ACP は、エージェント対応スタックの他の部分に代わるものではなく、焦点を当てたビジネス エクスペリエンス プロトコルとして最もよく理解されています。

## ACP が重要な場合

ACP は次の場合に関連します。

- 買い物客は AI の会話の中で商品を発見します
- 販売者は、不格好なリダイレクト ルートなしで支払いが行われることを望んでいます
- AI サーフェスが現実の販売チャネルになりつつある
- 販売者は、自動購入を行うのに十分なクリーンな製品データ、価格設定、ポリシー情報をすでに持っています。

この最後の点は過小評価されがちです。支払いによっては、間違った商品データは引き換えられません。 [ChatGPT 製品推奨 SEO チェックリスト](/es/docs/chatgpt-product-recommendations-seo/) では、選択する前に行う必要があることについて説明しています。

##トレーダーは最初に何を準備する必要がありますか?

|準備エリア |なぜそれが重要なのか |
|
---|
---|
|製品詳細 |エージェントは推奨または購入する前にデータが必要です |
|価格と在庫状況 |古いデータはすぐに信頼を失います |
|返品と配送 |購入者の制限には通常、次の両方が含まれます。
|不正行為の規制 |エージェント主導の注文には依然としてリスク管理が必要 |
|注文確認 |結果は決定的で監査可能でなければなりません。

ACP はチェックアウト プロセスの摩擦を軽減できますが、[実行層](/es/docs/execution-layer/) には依然として信頼性の高いバックエンド状態遷移が必要です。

## ACP と従来の電子商取引支払い |伝統的なボックス | ACPスタイルの流れ |
|
---|
---|
|購入者はウィンドウを閲覧します。購入者は AI インターフェイスに留まることができます |
|セッションと UI が支配的 |それらはプロトコルと構造化された状態を支配します。
|変換はページ フローに依存します |変換はコンテキストとデータ品質に依存します |
|人間はすべての詳細を読み取ります |エージェントはオプションを事前にフィルタリングできます。これが、古い有料分析だけでは不十分な理由です。販売者は、発見、推奨、転送、支払い完了、購入後の修正を単一のチェーンとして追跡する必要があります。

## リスクと限界

どのプロトコルも自動エントリとして扱わないでください。 ACP は以下に依存します。

- プラットフォームの採用
- 商用統合の品質
- ユーザーの信頼
- 正しい製品情報
- エージェントが買い手の代わりに行動する場合でも意味をなす支払いポリシー

規制された製品や高額な購入の場合、支払い画面がエージェントにネイティブになった場合でも、依然として人間による確認が必要な場合があります。

## よくある質問

### ACP はエージェント取引と同じですか?

いいえ、エージェント取引は市場における最も広範な変化です。 ACP は、AI エージェントをコンテキスト内で販売するためのプロトコル アプローチです。

### ACP は UCP に代わるものですか?

いいえ、ACP は支払いエクスペリエンスに焦点を当てていますが、UCP はより広範なビジネス ライフサイクルをカバーしています。

### すべての販売者は今すぐ ACP を実装する必要がありますか?

自動的ではありません。まず、AI 支援ショッピングがあなたのカテゴリにとって意味のあるチャネルであること、およびデータベースの準備ができていることを確認します。

### 最善の最初のステップは何ですか?

新しいプロトコル サーフェスを追加する前に、製品データ、製品スキーマ、ポリシーの明確さ、支払いの決定性を修正します。

## 結論

支払いが会話に近づいているため、ACP は重要です。 AI サーフェスを通じて販売したい販売者には、優れたコピー以上のものが必要です。明確な製品情報、信頼性の高いオペレーション、自動仲介用に設計された支払いフローが必要です。
