# BrowserGym と AutomationBench

BrowserGym と AutomationBench は、ページ ビューではなく、タスクを完了したかどうかによってエージェントがどのように評価されるかを示すため、AEO にとって重要です。エージェントと連携したい Web サイトは、ナビゲーション、API 検出、ポリシーの適用、最終状態の検証などの実際のワークフローをテストする必要があります。

## なんとハグ顔が出てきました

Hugging Face の記事を検索すると、Web エージェントおよびツールを使用するエージェントに関連する記事として [The BrowserGym Ecosystem for Web Agent Research](https://hf.co/papers/2412.05467) および [AutomationBench](https://hf.co/papers/2604.18934) が表示されました。

BrowserGym は、Web エージェントを評価するための標準化された環境に焦点を当てています。 AutomationBench は、検出、ポリシーの適用、データの正確性など、REST API を介したアプリケーション間のワークフロー オーケストレーションに重点を置いています。

この区分は AEO によく対応しています。

|基準点 | AEOレッスン |
|
---|
---|
|ナビゲータージム |エージェントが表示された Web フローを使用できるかどうかをテストします。 |
|自動化ベンチ |エージェントが API ワークフローを検出して実行できるかどうかをテストします。 |

## ベンチマークがトラフィックだけよりも優れている理由

AI の紹介やエージェントの検索は役立ちますが、タスクの成功を実証するものではありません。

|弱いメトリクス |より強力な AEO 指標 |
|
---|
---|
|ページビュー |タスクが完了しました |
|スクロールの深さ |状態遷移が成功しました |
| AI リファレンス |変換または検証結果 |
|ボット ユーザー エージェント |ツール呼び出しが成功しました |
|ページ滞在時間 |準拠完了 |

より広範な追跡については、[AEO KPI](/es/docs/aeo-kpis-measurement/) および [2026 年の AI エージェント Web トラフィック](/es/docs/ai-agent-web-traffic-2026/) を参照してください。

## サイトの小さな参照ポイントを作成します

始めるのに研究室は必要ありません。

1. 価値の高いタスクを 5 つ選択します。
2. 各タスクの正しい終了状態を定義します。
3. ブラウザ エージェントと、可能な限りツールを使用するエージェントを使用してタスクを実行します。
4. 失敗を記録します。
5. ラベル、ドキュメント、スキーマ、API、ポリシー、またはコミットを修正します。
6. 毎月繰り返します。

タスクの例:

|サイトの種類 |参照タスク |
|
---|
---|
|電子商取引 |製品を検索し、返品ポリシーを確認し、カートを準備します。 |
| SaaS |プランを比較し、デモをリクエストしてください。 |
|開発者ポータル |エンドポイント ドキュメントを検索し、テスト API 呼び出しを実行します。 |
|サポートサイト |請求に関する問題については、正しいサポート チャネルに問い合わせてください。 |
|旅行サイト |ポリシーの制限の下で返金可能な空室状況を確認します。 |

## 何を測定するか

1. 完了率。
2. 誤ったアクションの割合。
3. ポリシー違反率。
4. 回復の試行回数。
5. コンテキスト欠落エラー。
6. 人間による承認の移転。
7. 最終状態の検証。

これは、[エージェント評価パラメータ](/es/docs/agent-evaluation-benchmarks/) および [エージェントの可観測性と保護バリア](/es/docs/agent-observability-guardrails/) に直接リンクしています。

## よくある質問### BrowserGym と AutomationBench は SEO ツールですか?

いいえ。これらはエージェントの評価ベンチマークです。 AEO は単なる検索の可視性ではなく、エージェントのタスクの成功に関わるものであるため、AEO にとってこれらは重要です。

### Web サイトはどの参照スタイルを最初にコピーする必要がありますか?

フォームと UI フローを備えた Web サイトは、ブラウザー エージェントのテストから始める必要があります。多くの API を使用する企業は、API ツールとワークフローもテストする必要があります。### 合格または不合格ステータスとは何ですか?

承認ステータスは、チケットの作成、見積の成功、有効な API 応答、カートの準備、下書き予約の確認など、検証可能な結果です。

### エージェントのベンチマークはどのくらいの頻度で実行する必要がありますか?

ほとんどのサイトでは毎月で十分です。支払い、価格設定、サポート、または API ドキュメントが頻繁に変更される場合は、より頻繁に実行します。
