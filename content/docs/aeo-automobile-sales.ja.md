AI エージェントは、4 兆 2,600 億ドル規模の世界の自動車販売業界を書き換えています。消費者は車両の調査を代理店に委託し、数秒で 100 のモデルを比較します。フリート管理者は、数十の OEM の総所有コストを同時に評価する調達エージェントを使用します。アフターマーケット エージェントは、人間による見積もりを必要とせずに、自動的にメンテナンスのスケジュールを設定し、部品を調達します。

代理店消費のためにデータを構造化している OEM およびディーラー グループは、大幅に多くの潜在顧客を獲得します。人間中心の設定者や「見積依頼」フォームに依存するものは、エージェント主導の購入には見えなくなります。

## 自動車エージェントの 3 つのカテゴリー

### 構成および価格設定エージェント

購入者の要件に基づいて車両の候補リストを作成する、消費者対応のエージェント。仕様 (エンジン、バッテリー容量、航続距離、寸法、重量、積載量)、価格設定 (基本価格、オプション パッケージ、インセンティブ パッケージ、融資条件)、入手可能性 (生産スケジュール、ディーラー在庫、納期見積り)、および比較属性 (燃費、安全性評価、保証条件) など、各モデル バリアントの構造化データが必要です。

これらのエージェントはブランドを同時に比較します。コンフィギュレータ データが JavaScript を多用する対話型ツールとしてのみ存在する場合、エージェントはそれを抽出できません。各バリエーション、オプション、価格を機械可読形式で記載した構造化フィードが必要です。

### 法人および車両調達エージェント

フリート購入のために車両を評価する業者。顧客代理店には、消費者代理店が必要とするすべてのものに加えて、大量注文端末、フリート割引構造、テレマティクス統合仕様、メンテナンス契約条件、総所有コスト データ (減価償却費、燃料/エネルギーコスト、メンテナンス、保険) が必要です。

フリート調達は、[委任スペクトル](/es/docs/delegation-economy-agent-autonomy/) のレベル 3 に移行しています。つまり、事前承認された予算内で評価、選択、発注を行うエージェントです。ディーラーまたは OEM インフラストラクチャは、リード フォームだけでなく、確定的な注文 API をサポートする必要があります。

### サービスおよびアフターセールス代理店

購入後の車両のライフサイクルを管理する代理店。サービス プログラム、互換性データを含む部品カタログ、サービス予約のエンドポイント、リコール情報、保証ステータスの確認への構造化されたアクセスが必要です。

これらのエージェントは継続的に稼働し、メンテナンス スケジュールを監視し、サービス予約を積極的に予約します。静的なサービス ページではなく、リアルタイムのエンドポイントが必要です。

## AEO 実装パス

### 車両データの構造化

各車両構成を構造化データとして公開します。各仕様の明示的なフィールドを含む Schema.org Vehicle を使用します。 schema.org でカバーされていない車両固有の属性 (充電時間、特定の条件下での航続距離、牽引能力、荷台の長さ) については、カスタム JSON-LD を使用して拡張します。

リアルタイムで更新される機械可読な在庫フィードを作成します。エージェントは、在庫があるかどうかを確認できない車両を推奨しません。

### トランザクションエンドポイントエンドポイントを公開すると、特定のディーラーでのリアルタイム在庫の確認、正確な構成での見積もりの​​リクエスト、フリート購入の注文の送信、試乗のスケジュール、サービスの予約が可能になります。

各エンドポイントは決定的な応答を返す必要があります。 「担当者が連絡します」を返す見積もりエンドポイントはエンドポイントではありません。メインフォームです。エージェントは、定義された期間に有効な正確な構成の正確な価格を必要とします。

### インセンティブと資金調達のデータ

現在のインセンティブ スタックを構造化された形式で公開します。メーカーのリベート、ディーラーのインセンティブ、ロイヤルティ プログラム、融資率は頻繁に変更されます。現在のインセンティブ データにアクセスできないエージェントは不正確な比較を行い、競争上の地位を損なうことになります。

## 初期のユーザーからの結果

フル AEO スタックを採用している OEM および大規模ディーラー グループは、従来の Web のみの存在と比較して、エージェント主導のリードを 3 ～ 5 倍獲得しています。パーソナライズされた融資エージェントは、買い手を最適な融資構造に自動的に結び付けることで成約率を高めます。フリートの取得サイクルが数週間から数日に短縮されます。

## よくある質問

**AEO から最も恩恵を受ける自動車セグメントはどれですか?**
フリートの購入者はすでに構造化された評価基準を使用しているため、フリートの取得により最も早く投資収益率が得られます。購入代理店が成熟するにつれ、消費者向け自動車の販売は継続します。

**AEO は自動車ディーラーに取って代わるものですか?**完全にではありません。試乗、下取り交渉、資金調達に関する話し合いには、依然として人間同士のやり取りが役立ちます。エージェント主導の検出と初期セットアップにより、人間の作業負荷が軽減され、購入スケジュールが短縮されます。

**絶えず変化するインセンティブ データをどのように処理すればよいですか?**
毎日またはそれ以上の頻度で更新されるインセンティブ フィードを公開します。各インセンティブに有効期限を含めて、エージェントがデータの有効期限がいつ切れるかを把握できるようにします。インセンティブ データが古いと、エージェントが不正確な比較を行う原因になります。

**電気自動車と内燃機関車はどうですか?**
担当者は、バッテリー容量、さまざまな条件下での航続可能距離、充電速度、充電ネットワークの互換性、家庭用充電要件など、EV 固有のフィールドと両方の構造化データを必要とします。これらのフィールドは従来の自動車データ スキーマには存在しないため、追加する必要があります。**AEO を導入しない場合の競争上のリスクは何ですか?**
エージェントは構造化データから短いリストを作成します。あなたの車両がデータにない場合、それらはリストされません。エージェント主導のディスカバリーが増加するにつれて、AEO 対応ブランドと非 AEO 対応ブランドとの間のギャップが拡大します。
