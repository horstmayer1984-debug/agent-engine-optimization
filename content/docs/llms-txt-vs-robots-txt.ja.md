#llms.txt と robots.txt: AEO の実質的な違い

「robots.txt」はクロール アクセスを制御します。 「llms.txt」は、重要なコンテンツを厳選した機械可読マップを提供します。それらはお互いの代わりになるものではありません。 1つは追跡者に何を回復できるかを伝えます。もう 1 つは、エージェントまたは LLM ツールが、サイトが信頼できるものと最初に読む価値があると考えているものを理解するのに役立ちます。

## 簡単な比較

|アーカイブ |主な仕事 |典型的な読者 |ルールのステータス |
|
---|
---|
---|
---|
| `ロボット.txt` |クロール許可ルール |検索およびその他のトラッカー |主要なトラッカーによって解釈される確立されたプロトコル |
| `llms.txt` |選択したサイトと主要なリンクの概要 | LLM 指向のツールとエージェント |ポップアップコンベンション |

Google Robots のドキュメントでは、「robots.txt」のクロール制御機能について説明しています。 Chrome の Lighthouse エージェントのナビゲーション ドキュメントでは、「llms.txt」をポップアップ規約として説明しており、サイトがエラーなくファイルを配信しているかどうかを監査するようになりました。

- [Google ロボットのドキュメント.txt](https://developers.google.com/crawling/docs/robots-txt/robots-txt-spec)
- [Chrome Lighthouse: llms.txt 監査](https://developer.chrome.com/docs/lighthouse/agentic-browsing/llms-txt)
- [llms.txt プロジェクト サイト](https://llmstxt.org/)

## robots.txt ファイルは何のためにありますか?

クローラーにアクセスできる URL パスを指示する必要がある場合は、「robots.txt」を使用します。典型的な例:

- ステージングフォルダーをロックする
- 内部検索結果をクロール パスから遠ざける
- 特定のユーザーエージェントを許可または禁止します
- クローラーにサイトマップをポイントさせる

これはインフラストラクチャ レベルのガイドです。どのページが製品概要に最適であるか、どの価格ページが正規であるか、またはどのプロトコル ガイドを最初に信頼すべきかについては説明されていません。

## llms.txt は何のためにありますか?

サイトの最も重要なリソースの簡潔で選択された概要が必要な場合は、「llms.txt」を使用します。通常、有用なファイルには次のものが含まれます。

- 場所は何ですか
- 中心的な概念
- 最高のホームページ
- 重要な文書またはプロトコルのリファレンス
- サイトマップなどの機械可読リソース

[llms.txt ガイド](/es/docs/programming-llms-txt/) には実装の詳細が記載されています。 [Lighthouse エージェント ナビゲーション ガイド](/es/docs/lighthouse-agentic-browsing-audit/) では、ブラウザ ツールがそれを探し始めた理由について説明しています。

## なぜ人々はそれらを混同するのでしょうか

ファイル名は似ており、どちらもドメインのルートにあります。しかし、彼らは異なる質問に答えます。

|質問 |正しいファイル |
|
---|
---|
|このボットは `/private/` をクロールできますか? | `ロボット.txt` |
|エージェントは最初にどの文書を読む必要がありますか? | `llms.txt` |
|サイトマップはどこにありますか? |通常は `robots.txt` ですが、`llms.txt` にリンクされることもあります。
|サイトの正規の定義は何ですか? | `llms.txt` またはホームページ |
|このファイルは認証を置き換えることができますか? |なし |

## AEO導入パターン

エージェント対応 Web サイトの場合は、次の両方を使用します。1. 「robots.txt」を有効な状態に保ち、ビジネス ポリシーに合わせてください。
2. 重要なページのみを含む簡潔な `llms.txt` を公開します。
3. 製品、プロトコル、および価格設定のページは、サイト アーキテクチャと関連する「llms.txt」の両方からリンクされたままにしておきます。
4. テンプレートまたは CDN に大きな変更を加えた後、再テストします。
5. API、MCP、UCP、または同様のプロトコルを介してアクション サーフェスを個別に追加します。

最後の点が重要です。サイトに完全なテキスト ファイルが含まれていても、[ランタイム レイヤー](/es/docs/execution-layer/) が存在しない場合はエージェントが使用できない場合があります。

## よくあるエラー- `llms.txt`をトラッカー権限ファイルとして扱います
- 「llms.txt」に各サイトの URL を入力します
- 「robots.txt」がコンテンツ ガバナンスを解決すると仮定します
- ピラーページが変更されたときに「llms.txt」を更新するのを忘れる
- 便利な HTML ページの代わりとして任意のファイルを使用します

## よくある質問

### llms.txt は新しい robots.txt ですか?

いいえ。「robots.txt」はクロール権限のためのものですが、これは厳選された発見支援として最もよく理解されています。

### 両方のファイルが必要ですか?

サイトが通常の検索可視性とエージェントが読み取り可能な検出を必要としている場合は、そのとおりです。それらはさまざまな機能を果たします。

### llms.txt は AI システムが私のコンテンツを使用できるかどうかを制御しますか?

それ自体ではありません。アクセスの決定には、追跡ポリシー、法的条件、および技術的制御を使用します。

### すべてのページを llms.txt に含める必要がありますか?

いいえ。アーカイブは、サイトについて最もよく説明されているいくつかのページを強調表示する場合に最も役立ちます。

## 結論

クロールを制御するには「robots.txt」を使用します。曖昧さを軽減するには「llms.txt」を使用してください。 AEO には両方が必要ですが、どちらも明確なページ、強力な内部リンク、実際のエージェント機能に代わるものではありません。
