# UX および AEO SEO エージェント

最新のページは人間、検索クローラー、AI エージェントという 3 つのユーザーにサービスを提供するため、エージェント UX は AEO SEO に影響します。アクション、ポリシー、フォーム、または確認があいまいな場合、ページはランク付けされてもエージェントに不合格となる可能性があります。エージェントの優れたユーザー エクスペリエンスには、セマンティック HTML、構造化データ、明確なステータス、および危険なアクションに対する人間の承認が使用されます。

## UX が AEO SEO に属する理由

従来の SEO では、UX をページ速度、モバイルの使いやすさ、コンテンツの明瞭さとして扱うことがよくあります。 AEO SEO は、UX を拡張してマシン ユーザーを含めます。

AI エージェントは以下を理解する必要があります。

- ページの内容
- どのようなアクションが可能ですか
- どのような制限が適用されるか
- 人間の承認が必要な場合
- アクションが成功したか失敗したか

既存のページ [エージェント UX とヒューマンインザループ設計](/es/docs/agent-ux-human-in-the-loop/) では、インターフェイス パターンについて説明しています。このページでは、その取り組みを検索の可視性と AEO に結び付けます。

## 人間の UX とエージェントの UX

|ユーザーエクスペリエンス要素 |人間の目標 |エージェントの目的 |
|
---|
---|
---|
|タイトルの階層 |ページをスキャン |セクションのテーマと意味を推測する |
|製品表 |視覚的に比較 |属性の抽出 |
|ボタン |クリアアクションをクリックします |安定した銘柄を特定する |
|フォーム |情報を送る |必須フィールドを理解する |
|エラーメッセージ |手動で回復する |次のステップのルート |
|確認ページ |ユーザーを安心させる |タスクの完了を確認する |

Google の初期の SEO ガイドでは、コンテンツを有用でアクセスしやすいものにすることに重点を置いています。 Agent UX は、そのアイデアを機械可読アクションに拡張します。

## セマンティック HTML が最初の解決策です

実数を使用します:

- ヘッダー
- リスト
- テーブル
- ボタン
- ラベル
- 形状
- リンク
- ステータスメッセージ

「div」スタイル要素や画像のみのテキストなどの重要なコンテンツを作成しないでください。アクセシビリティの実践は、どちらも明示的な構造を必要とするため、エージェントに役立ちます。

## 人間参加型の設計

すべてのタスクが自律的である必要はありません。高リスク株は検討のために停止する必要があります。

- しきい値を超える支払い
- アカウントの変更
- キャンセル料が発生する予約
- 医学的、法的、または財務上の決定
- 取り消し不能なアクション

AEO SEO の場合、エージェントはどのアクションに確認が必要かを知る必要があるため、これは重要です。ページ、フォーム、および機械可読ドキュメントからこれらのルールをリンクします。

## 導入チェックリスト

1. メイン コンテンツをサーバー上でレンダリングするか、クロール可能な HTML で利用できるようにします。
2. 比較、形状、アクションの意味要素を使用します。
3. アクションに簡単な言葉でラベルを付けます。
4. アクションの近くに制約を表示します。
5. 構造化されたエラー メッセージと成功メッセージを追加します。
6. 可能な限り、安定した URL を含む確認ページを使用します。
7. 文書承認基準。
8. エージェントの失敗に対するフィードバック パスを追加します。 [AEO 開発者ガイド](/es/docs/developers-guide-aeo/) と [AI エージェント プログラミング Web サイト](/es/docs/programming-websites-for-ai-agents/) では、実装について詳しく説明されています。

## よくある質問

### エージェントの UX はランキングを向上させますか?

間接的に。タスクの明確さ、アクセシビリティ、構造化されたコンテンツ、および信頼性を向上させ、SEO、AI 検索、AEO をサポートします。

### エージェントの UX にはセマンティック HTML で十分ですか?

それがベースです。複雑なアクションには、API、MCP ツール、明確なポリシー、およびコミット状態が必要な場合もあります。### それぞれのアクションは自律的である必要がありますか?

いいえ。リスクの高い行動には人間の承認または特定の権限が必要です。

### エージェントの UX の最速の改善は何ですか?

曖昧なアクション タグ、隠しフィールド、スクリプトのみのコンテンツを、明確なセマンティック HTML、目に見える制約、構造化されたコミットに置き換えます。
