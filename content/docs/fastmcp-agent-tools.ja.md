# FastMCP エージェント ツール

FastMCP は、API とビジネス オペレーションを MCP ツールに変換する際の負担を軽減するため、AEO にとって重要です。エージェントがページをスクレイピングしたりフォームを推測したりする代わりに、構造化ツールを呼び出すことができれば、Web サイトとアプリケーションはより信頼性の高い実行ターゲットになる可能性があります。

## FastMCP が関連する理由

GitHub プラグインにより MCP ツールが明らかになり、2026 年 6 月 28 日にチェックされた GitHub API メタデータには、25,000 個を超えるスターが付いた [PrefectHQ/fastmcp](https://github.com/PrefectHQ/fastmcp) が示されました。このプロジェクトは、Python での MCP サーバーとクライアントの作成に焦点を当てています。

FastMCP は戦略に代わるものではありません。どのアクションをツールに変換する必要があるかをチームが把握したら、それが実装パスとなります。

プロトコルの背景については、[MCP リソース対ツール対プロンプト](/es/docs/mcp-resources-tools-prompts/) および [MCP 対エージェント API](/es/docs/mcp-vs-api-for-agents/) を参照してください。

## MCP ツールになるべきものは何ですか?

| Web サイトまたはアプリのタスク | MCP ツールの候補として適していますか? |理由 |
|
---|
---:|
---|
|ドキュメントの検索 |はい |リスクが低く、実用性が高い。 |
|製品の在庫状況を確認する |はい |エージェントには最新の構造化データが必要です。 |
|サポートチケットを作成する |はい、検証あり |状態は変化しますが、制限される場合があります。 |
|注文する |承認がある場合のみ |身元確認、支払い、監査チェックが必要です。 |
|アカウントを削除する |一般的にはそうではありません |リスクの高い取り返しのつかない行為。 |

## ツールを構築する前の AEO チェックリスト

1. 簡単な言語でアクションを定義します。
2. 必要な入力を定義します。
3. 出力とエラー状態を定義します。
4. 認証ルールと認可ルールを追加します。
5. レート制限を追加します。
6. 危険な行為に対して人間の承認を追加します。
7. ログとトラッキング ID を追加します。
8. 「llms.txt」または開発者センターからツールのドキュメントをリンクします。

ここで、[実行層](/es/docs/execution-layer/) が役に立ちます。

## MCP ツールと Web サイト ページの比較

|フロントページの焦点 |ツールファーストのアプローチ |
|
---|
---|

ほとんどのサイトでは両方が必要です。ページでは説明しています。ツールが実行されます。

## 導入前にツールコントラクトを作成します。

FastMCP を通じてツールを公開する前に、平易な言語でツールを定義します。動作、必須の入力、オプションの入力、認可範囲、出力スキーム、副作用、エラー コード、アクションが元に戻せるかどうかを示します。プロモーターにとって契約が曖昧であれば、エージェントにとっても曖昧になります。読み取り操作を状態変更操作から分離してください。 「get_order_status」と「cancel_order」は広範なツールのモードであってはなりません。スタンドアロン ツールを使用すると、ユーザーの権限、登録、再試行、承認についての推論が容易になります。

無効な入力は、成功した呼び出しと同じくらい慎重にテストしてください。エラーでは、フィールド、失敗したルール、および修正されたデータでの再試行が許可されているかどうかを識別する必要があります。エージェントが結果を確認するために識別子またはステータスを必要とする場合は、一般的な成功メッセージを返さないようにします。

## よくある質問

### MCP には FastMCP が必要ですか?いいえ、これは一般的な展開パスです。 MCP は、他の SDK およびサーバーとともにデプロイできます。

### すべての API を MCP ツールにする必要がありますか?

いいえ、制限された高価値の操作を最初に公開します。ガバナンスが整うまでは高リスク株は避けてください。

### これは AEO にどのように役立ちますか?

Web サイトまたはサービスを発見して理解した後、エージェントに構造化された実行パスを提供します。

### マーケターは何を知っておくべきですか?

変換がフォーム、見積もり、予約、またはサポート アクションに依存する場合、技術チームはより優れたコピーだけでなく、ツール インターフェイスを必要とする場合があります。
