# ユニバーサル カートとエージェント コマース: 販売者向け準備ガイド

ユニバーサル カートは、ショッピング カートを販売者所有のセッションからエージェント支援のクロスショッピング レイヤーに移動します。販売者にとってのリスクは、支払いページが表示されなくなることだけではありません。最大の変更点は、製品データ、カート ルール、ID、ポリシー、チェックアウトが、エージェントが自信を持って購入アイテムを追加、比較、ルーティングできるかどうかに影響を与えるようになったことです。

## ユニバーサル カートの変更点

Google は、エージェント コマース推進の一環として、Google I/O 2026 で [ユニバーサル カート](https://blog.google/products-and-platforms/products/shopping/google-shopping-cart/) を紹介しました。 Googleによると、カートはすべてのGoogleサービスと小売店で機能するように設計されており、UCPはカートからのチェックアウトや販売者のサイトへの商品の転送を支援するという。

つまり、カートは店頭のボタンというよりは、永続的なインテント コンテナに近いということです。商品は、検索、Gemini、ショッピング、広告、または Future Retailers からカートに入れることができます。

これは、[エージェント取引の 5 つのレベル] (/docs/five-levels-agentic-commerce/) およびより広範な [ユニバーサル取引プロトコル ガイド] (/docs/universal-commerce-protocol/) に直接つながります。

## 車が AEO 表面になった理由

従来の e コマースでは、販売者が商品ページ、カート、アップセル、配送オプション、チェックアウト ルートを制御します。エージェント取引では、これらの決定の一部は上流で行われます。

|伝統的なカート |ユニバーサルカートまたはエージェント |
|
---|
---|
|商業施設に住んでいます |表面や小売店に追加可能 |
|人間の変換用に最適化 |それはエージェントと人間にとって理解できるものでなければなりません |
|サイト固有のセッション ロジックを使用します。商品、価格、ポリシーに関するポータブル データが必要 |
|販売者がすべてのステップを制御します。エージェントまたはプラットフォームが旅行の一部を提供する場合があります |
|帰属はページセッションに基づいています |帰属はエージェント/参照のコンテキストに依存する場合があります。

[エージェント エンジンの最適化] (/docs/what-is-aeo/) の場合、これは、カートが単なる UX ではなく、実行層の一部であることを意味します。

## トレーダーの準備チェックリスト

トレーダーは以下を準備する必要があります。

1. フィード、スキーマ、および内部カタログ ID と一致する製品 ID。
2. サイズ、色、パッケージ、サブスクリプション、地域を区別するバリアント データ。
3. リアルタイムの価格と在庫シグナル。
4. エージェントが分析できる配送、税金、返品ポリシーのフィールド。
5. UCP 互換のカート API またはカート機能。
6. 登録加盟店の明確な行動。
7. 確実な確認、キャンセル、返金ステータス。
8. エージェントの送信元トラフィックを通常のセッションから分離する分析。

ページ [AI 購買エージェント向けの構造化製品データ] (/docs/product-structured-data-ai-shopping/) は製品レイヤーをカバーしています。このページではカート層について説明します。

### ユニバーサル カートは支払いと同じですか?

いいえ、ユニバーサル カートはショッピング コンテナです。支払いは、統合とユーザーの選択に応じて、Google がサポートするフローを通じて、または販売者のサイトを通じて行うことができます。

### Universal Cart は販売者の Web サイトに代わるものですか?

いいえ、販売業者のサイトには依然として商品データ、ポリシー、信頼シグナル、顧客サポート、および多くの支払いフローが含まれています。ただし、カートはサイトの外に形成することもできます。

### トレーダーは最初に何をすべきでしょうか?

エージェント向けの高度なチェックアウト フローを追求する前に、カタログの品質、製品レイアウト、在庫の精度、ポリシーの明確さ、カート API の一貫性を修正します。

### これは SEO にどのような影響を与えますか?

SEOはAEOに拡張されます。製品ページをランク付けするだけではもはや十分ではありません。エージェントは、カートに追加して確認できる信頼性の高いデータを必要としています。
