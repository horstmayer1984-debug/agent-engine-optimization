# Universal Trade Protocol (UCP): 実用的なガイド

ユニバーサル コマース プロトコルは、エージェント主導のショッピングのために Shopify と Google が共同開発したオープン コマース標準です。これは、販売業者や代理店が、新しい AI サーフェスごとにカスタム統合を再構築することなく、機能を取引し、標準化された業務運営を処理し、発見、支払い、注文、購入後などのワークフローを実行できるように設計されています。

## UCP は何をしますか?

Shopify は、UCP をエージェントとコマースを統合するためのプロトコルとして説明しています。その公式資料では、普遍的なプリミティブ、標準化された操作、カスタム拡張、動的なネゴシエーション、オープンな支払い、輸送の柔軟性が強調されています。

| CPU 容量 |なぜそれが重要なのか |
|
---|
---|

## トレーダーが注意すべき理由

従来の e コマース統合は、店頭がメイン インターフェイスであることを前提としています。エージェント取引はそれを変えます。販売者は、割引、配送、支払い、フルフィルメントに関して同じルールを維持しながら、検索、チャット、ウォレット、またはアシスタントから購入者にサービスを提供する必要がある場合があります。

UCP が重要なのは、各マーチャントにエージェント プラットフォームごとに固有のコネクタの作成を強制するのではなく、マーチャント層の標準化を試みるためです。

[エージェント取引ガイド] (/docs/agentic-commerce/) では戦略的転換について説明しています。 [貿易エージェント実行ガイド] (/docs/agentic-commerce-execution/) では、発見後に必要なバックエンド作業について説明します。

## UCP と通常の API

|通常の API 統合 | UCP アプローチ |
|
---|
---|
|販売者固有のエンドポイント |共有コマースのセマンティクス |
|オーダーメイドの発見 |能力の交渉 |
|支払いの前提条件が組み込まれることがよくあります。オープンペイメントハンドラー |
|パートナーによる再構築 |互換性のあるエージェント間での再利用 |
|実装にリンクされたトランスポート |輸送用フレキシブルモデル |

UCP によって、優れた API が不要になるわけではありません。共有言語を使用した取引操作が提供されるため、エージェントはトレーダーがサポートしている内容を推論できます。

## トレーダーが準備すべきもの プロトコルを実装する前に、基本を正しく理解してください。

1.正確な製品カタログ
2. 現在の価格と在庫状況
3. 決定的なカートとチェックアウトのロジック
4. 配送規定と税金
5. 返品およびキャンセルポリシー
6. 注文ステータスの可視性
7. 自動化されたアクションの監査可能性

基礎となるデータがあいまいな場合、ブローカーは安全に取引を完了できないため、その準備は [AEO e-commerce guide](/es/docs/aeo-ecommerce/) と大きく重複します。

## よくある質問

### UCP は Shopify 販売者のみを対象としていますか?

いいえ、Shopify は、Google と共同開発したオープン プロトコルとしてこれを提示し、より広範なビジネス エコシステムにオープンです。

### UCP は支払い処理業者に代わるものですか?

いいえ。Shopify では、単一の必須処理業者ではなく、オープンな支払いと交渉による支払いハンドラーについて説明します。

### UCP は支払いにのみ適用されますか?

いいえ。公式資料には、証拠開示、支払い、注文、購入後のフローが含まれます。

### トレーダーは最初に何をすべきでしょうか?

プロトコルの統合を検討する前に、製品、価格設定、在庫状況、注文ルールを機械で読み取り可能にしてください。

## 結論

UCP は、取引を単一の統合の問題から再利用可能なプロトコルの問題に変えるため、重要です。すでにクリーンな製品データと決定論的なオペレーションを持っている販売者は、チャネルの需要が実際に高まったときにそれらを採用するのに有利な立場にあります。
