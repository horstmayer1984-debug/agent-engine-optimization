# 従来の CMS はヘッドレス CMS より高速ですか?

従来の CMS は、効率的でキャッシュされ、軽くカスタマイズされている場合、ヘッドレス CMS よりも高速になります。ヘッドレス CMS は、インターフェースが静的に生成され、エッジでキャッシュされ、JavaScript でオーバーロードされていない場合に高速になります。アーキテクチャだけが速度を決めるわけではありません。実装はそうです。

## 正直な答え

この質問が人気があるのは、ヘッドレス CMS ベンダーが速度についてよく語る一方で、WordPress やその他の従来の CMS プラットフォームは依然として非常に高速である可能性があるためです。

速度は以下によって決まります。

- 宿泊施設
- キャッシング
- レンダリング
- テーマまたはインターフェースコード
- 画像の最適化
- サードパーティのスクリプト
- データベースクエリ
- 構築および実装戦略

より広範なトレードオフについては、[ヘッドレス CMS と従来の CMS](/es/docs/headless-cms-vs-traditional-cms/) を参照してください。

## パフォーマンスの比較

|係数 |従来の CMS |ヘッドレスCMS |
|
---|
---|
---|
|初期設定 |優れたホスティングを使用すると、非常に高速になることがよくあります。それはインターフェースの構造によって異なります。
|静的配信 |キャッシュ/プラグインで可能 |静的生成と共通 |
| JavaScript の重み |テーマ/プラグインによって異なります |フレームワーク/コンポーネントに依存 |
|画像管理 |プラグインまたはプラットフォームによって異なります | CDN イメージまたは依存フレームワーク |
|編集プレビュー |一般にセットアップは簡単です |構築する必要があります |
|キャッシュの無効化 |プラグインによって管理されることが多い | CMS とフロントエンドを接続する必要があります |

## 従来の CMS の方が高速な場合

従来の CMS は、次の場合に高速になります。

- テーマは光です
- プラグインスタックが小さい
- フルページキャッシュが有効になっています
- 画像は最適化されています
- ホスティングが強力です
- ページはほとんどが静的です

小規模なサービス Web サイトの場合、効率的な従来の CMS の方が、複雑なヘッドレス インターフェイスよりも高速で安価な場合があります。

## ヘッドレス CMS の方が高速な場合

ヘッドレス CMS は、次の場合に高速になります。

- ページは静的に生成されます
- アセットは CDN から提供されます
- JavaScript は小さいままです
- 画像は自動的に変換されます
- コンテンツ API はキャッシュされます
- インターフェイスにより、不必要なクライアント側の作業が回避されます

Next.js のドキュメントには、静的生成によりビルド時にページを事前レンダリングし、CDN 経由で提供できると記載されています。これは、ドキュメント、ブログ、ランディング ページ、および多くのコンテンツ ハブにとって確実なパターンです。

## SEO パフォーマンスのリスク

|リスク |影響を受けるアーキテクチャ | SEO への影響 |
|
---|
---|
---|
|重いプラグイン |従来の CMS |遅いページとレンダリングのブロック リソース |
|強力な水分補給 |ヘッドレスCMS | JavaScript の増加とインタラクションの速度の低下 |
|最適化されていない画像 |両方 |最悪の LCP と帯域幅 |
|キャッシュエラー |両方 | TTFBが遅い |
|クライアント向けの独占コンテンツ |ほとんど頭なし |クロールとインデックス作成のリスク |
|サードパーティのラベル |両方 | Poor Basic Web Vitals |[Headless SEO](/es/docs/headless-seo/) ガイドでは、速度とクローラビリティを同時にテストする必要がある理由を説明しています。 CMS 固有の実装の詳細については、[ヘッドレス CMS SEO チェックリスト](/es/docs/headless-cms-seo/) を使用してください。

## 何を測定するか

ラボツールと現場データを使用して測定:- 基本的なウェブバイタル
- LCP要素
- JavaScriptのパッケージサイズ
- TTFB
- キャッシュヒット率
- 画像転送サイズ
- レンダリングロックリソース
- ハイドレーション前の完全な HTML。

抽象的に「WordPress」と「headless」を比較しないでください。実際のページを比較してください。

## よくある質問

### ヘッドレス CMS は常に高速ですか?

いいえ、重いヘッドレス インターフェイスは、十分にキャッシュされた従来の CMS よりも遅くなる可能性があります。

### WordPress はデフォルトでは遅いですか?

いいえ。WordPress のパフォーマンスは、テーマ、プラグイン、ホスティング、キャッシュ、メディア処理によって大きく異なります。

### Google は高速なサイトをより良くランク付けしますか?

ページエクスペリエンスとコアウェブバイタルは重要ですが、スピードもSEOの一部です。コンテンツの関連性、リンク、意図の一致、技術的なアクセシビリティも重要です。

### SEO ページの最も安全なパフォーマンス戦略は何ですか?

完全な HTML を配信し、画像を最適化し、JavaScript を削減し、キャッシュを使用し、リリース前に実際のテンプレートをテストします。
