x402 は、MCP (ツール呼び出し) と A2A (エージェント間通信) に支払いを追加することで AEO スタックを完成させます。 4 つのレイヤーがすべて整っていると、エージェントは llms.txt 経由でサービスを発見し、MCP 経由でツールを呼び出し、A2A 経由で共同作業し、x402 経由で即時支払いを行うことができます。どの段階においても人間の介入はありません。

## 4 層 AEO スタック

### レイヤ 1: ディスカバリー

llms.txt とエージェント カードは、サイトが提供するもの、機能の場所、サポートされているプロトコルをエージェントに伝えます。ここがエントリーポイントです。これがないと、エージェントはあなたの存在を知りません。

### レイヤ 2: 実行

MCP サーバーは、エージェントが動的に検出して呼び出すことができる記述されたツールとして取引アクションを公開します。これは、エージェントが読み取りからアクションに至る [実行層](/es/docs/execution-layer/) です。

### レイヤ 3: コラボレーション

A2A プロトコルにより、エージェント間のタスク共有が可能になります。エージェントはサブタスクをそのサービスに委任します。サービスはそれを処理し、構造化された結果を返します。これにより、サイトは単なる受動的なエンドポイントではなく、マルチエージェント ワークフローの共同参加者になります。

### レイヤ 4: 支払い

x402 では、HTTP レベルでの即時マイクロペイメントが可能です。ツールの呼び出しやタスクの完了はリクエストごとに収益化できます。エージェントは USDC で支払い、サーバーはオンチェーンで検証し、アクセスが許可されます。

各レイヤーは前のレイヤーの上に構築されます。実行を伴わない発見は情報です。協力なしでの実行は孤立です。支払いのないコラボレーションは無料です。

##フルスタックインタラクションの仕組み

エージェントにはプレミアム市場分析データが必要です。 MCP レコードを検索し、エージェント カードを通じてサービスを見つけます。価格と機能の詳細については、llms.txt をお読みください。

エージェントは、必要なパラメータを指定して市場分析 MCP ツールを呼び出します。サーバーは価格を含む 402 応答を返します。エージェントは X-PAYMENT ヘッダーを構築し、Base で 0.05 USDC を支払い、再試行します。サーバーは支払いを検証し、分析を実行して結果を返します。

エージェントがマルチエージェント ワークフローの一部である場合、A2A タスク共有を介して結果を次のエージェントに渡します。サービスは、[フィードバック ループ](/es/docs/agent-feedback-loops/) を改善するためにインタラクションを記録します。

合計経過時間: 5 秒未満。エージェントの総コスト: 0.05 ドルとごくわずかな取引手数料。あなたへの収益: リクエストごとに 0.05 ドル、リアルタイムで決済されます。

## 各レイヤーが他のレイヤーを必要とする理由

x402 を使用しない MCP は、エージェントがツールを無料で使用することを意味します。使用量は得られますが、収益は得られません。これは見込み客発掘には機能しますが、プレミアム サービスには機能しません。 MCP を使用しない x402 は、エージェントが料金を支払うことはできますが、ツールを検出したり呼び出したりするための標準化された方法がないことを意味します。支払いレイヤーがあり、それを通じて販売するものは何もありません。

MCP と x402 を使用しない A2A は、エージェントがタスクを要求できるものの、標準のツール インターフェイスや支払いメカニズムがないことを意味します。実行や支払いのないコラボレーションは、単なるメッセージングです。

フルスタックにより、完全な自律取引システムが構築されます。発見は実行につながります。実行はコラボレーションにつながります。支払いによりサークルは終了します。

##比較: 部分スタックとフルスタック|レイヤー |この層がなければ |このレイヤーを使用すると |
|---|---|---|
|ディスカバリー (llms.txt、エージェント カード) |エージェントがあなたを見つけられません |エージェントは何度も検索してあなたを発見します |
|実行 (MCP) |エージェントは読み取りますが、行動できません |エージェントはツールを直接呼び出します。
|コラボレーション（A2A） |各エージェントの対話は分離されています。エージェントはマルチエージェント ワークフローでタスクを委任します。
|支払い (x402) |無料で使用またはサブスクリプション |リクエストによる収益化、ゼロフリクション |

## 導入シーケンス

まずは検出から始めます。llms.txt と Agent-card.json を公開します。これには 1 日かかりますが、費用はかかりません。

実行の追加: 上位 3 ～ 5 つのツールを使用して MCP サーバーを作成します。既存の SDK ではこれに 2 ～ 4 時間かかります。

支払いの追加: プレミアム エンドポイントに x402 ミドルウェアを展開します。事前に構築されたミドルウェアを使用すると、これには 1 ～ 2 時間かかります。

コラボレーションの追加: マルチエージェント ワークフロー用の A2A タスク エンドポイントを公開します。複雑さに応じて、これには 1 ～ 2 日かかります。

フルスタックへの総投資は、既存の API エンドポイントを備えたサイトの開発時間にすると 1 週間未満です。 [AEO 実装ガイド] (/docs/implement-aeo/) では、そのパスを段階的に説明しています。

## よくある質問

**4 つのレイヤーすべてが必要ですか?**
検出と実行 (レイヤー 1 および 2) から始めます。プレミアム エンドポイントの支払い (レイヤー 4) を追加します。マルチエージェントのワークフローに参加する場合は、コラボレーション (レイヤー 3) を追加します。

**最初にどのレイヤーを実装すればよいですか?**
検出 (llms.txt およびエージェント カード)。これは実装が最も速く、他のすべての前提条件です。

**MCP なしで x402 を使用できますか?**
はい、標準の REST エンドポイントを使用します。ただし、MCP では、カスタム統合を行わなくても、サポートされているエージェントが有料ツールを検出できるようにしています。組み合わせの方が強いですね。

**フルスタックではどれくらいの収益が得られますか?**
交通量と料金によって異なります。 1 リクエストあたり 0.01 ドルで 1 日あたり 10,000 のエージェント リクエストを処理するプレミアム データ エンドポイントは、1 日あたり 100 ドル、または 1 か月あたり約 3,000 ドルを生み出します。そこから価格と量を調整します。

**小規模サイトにとってフルスタックは過剰ですか?**エージェントが料金を支払うようなプレミアム コンテンツやサービスがある場合はそうではありません。導入コストは低め (1 週間未満) で、インフラストラクチャはトラフィックに応じて自然に拡張されます。
