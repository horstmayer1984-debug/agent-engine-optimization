#x402 vs ACP vs MPP: 支払いレイヤー、有料レイヤー、自動支払いレイヤー

x402、ACP、MPP は競合他社とみなされることが多いですが、それぞれ異なる支払い問題を解決します。 x402 は、HTTP リソースの軽量決済パターンです。 ACP は、エージェントが開始する購入のための販売者支払いプロトコルです。 MPP は、セッション、従量制使用量、および継続的なエージェント アクティビティに対する自動支払い標準です。

それらを比較する最も簡単な方法は、エージェントが何を購入しているかを尋ねることです。

エージェントが API 応答を購入する場合は、x402 が適合します。エージェントが販売者から製品を購入する場合、ACP が適合します。エージェントが進行中のタスク中に繰り返し支払いを行う場合、MPP はより興味深いものになります。

## 中央の違い

|質問 | x402 | ACP | MPP |
|---|---|---|---|
|何が解決されているのでしょうか? |保護された HTTP リソースの支払い |エージェントが支払いを開始しました |時間の経過とともに自動支払い |
|主要購入者のパターン | API クライアントまたはエージェント |ユーザーを代理する購入エージェント |エージェント、サービス、またはマシンのワークフロー |
|ベストバリューユニット |リクエストまたはリソース |注文またはカート |セッション、ストリーミング、サブスクリプション、従量制使用量 |
|支払いモデル |一般的に安定したコインの清算 |販売者のチェックアウトとチェックアウト資格情報 |自動支払いセッションによる法定通貨または仮想通貨 |
|最良の例 | API 呼び出しに対して 0.01 USDC を支払う | AI アシスタント内で製品を購入する | API の使用料を継続的に支払う |

## x402 は何が得意ですか?

x402 は、プロトコルを Web に近い状態に保つことができるため、魅力的です。サーバーは HTTP 402 Payment Required で応答し、価格と支払い要件を指定して、リソースを提供する前に支払いを検証できます。

エージェントのワークフローでは、これが可能です。小規模なデジタル トランザクションからのアカウント設定、サブスクリプションのネゴシエーション、および API キーのプロビジョニングを排除します。

x402 の適切な使用例は次のとおりです。

1. 有料MCPツール
2. プレミアム データ エンドポイント
3. リクエストによる推論
4. 機械可読レポート
5. エージェントがアクセス可能なコンテンツ ペイウォール
6. 実行前に価格がわかっている API 呼び出し。

弱点は技術的な優雅さではありません。弱点は範囲です。 x402 は、小売行程、カートのステータス、返品、税金、フルフィルメント、または顧客サービス全体を説明するものではありません。

## ACP が得意とすること

ACP はエージェント取引用に設計されています。 OpenAI と Stripe は、AI アシスタントがユーザーが販売者を発見、評価し、購入するのを支援するケースのためにこれを開発しました。 Stripe のエージェント コマース ドキュメントは、共有支払いトークン、製品カタログ管理、およびインコンテキスト販売を対象としています。それは x402 とは別の問題です。小売での購入は単なる支払いではありません。在庫、バリエーション、配送、税金、登録販売者の質問、注文確認、キャンセル、返品が含まれます。

ACP は、エージェントが消費者の購入フロー内で動作する場合に強力になります。

## MPP が得意とすること

Stripe と Tempo が提供する MPP は、エージェントが一度も電話をかける必要のない自動支払い向けに設計されています。セッションの下で操作したり、リソースの使用を継続したり、作業の継続に応じて繰り返し支払いを行ったりすることができます。

それは自律システムに適しています。エージェントは、今後のコールの正確な数を常に知っているわけではありません。見積もり、期間、料金上限、およびジョブ完了後の調整が必要になる場合があります。

MPP は概念的に次のものに最もよく適合します。1. APIの継続利用
2. エージェントのサブスクリプション
3. 従量制課金
4. ストリーミングマシンによる支払い
5. 自律的なワークフローのための委任された予算

## 3 人で協力する方法

現実的なスタックでは、複数のプロトコルを使用できます。

AI エージェントは、調査ステップ中にマーチャント サーフェスを通じてマーチャントを発見し、ACP を使用して支払いを行い、AP2 またはカード ネットワーク制御に基づいて承認を行い、x402 または MPP を通じてサードパーティ データ プロバイダーに支払いを行うことができます。

開発者のワークフローでは、エージェントは分離されたリクエストに対して x402 経由で有料の MCP ツールを呼び出し、繰り返し使用する長時間実行のデータ タスクのために MPP に切り替えることができます。

重要な設計ルールは関心事の分離です。

## 開発者はどれを最初に実装すべきでしょうか?

|あなたは | を操作します| から始める理由 |
|---|---|---|
|有料 API | x402 |エンドポイントは価格を設定し、各リクエストを検証できます。
|繰り返し使用できる SaaS API | MPP |セッションおよび使用量ベースの請求事項 |
|電子商取引ストア | ACP |支払いプロセスが本当の問題です |
|データ市場 | x402 と MPP |一度購入される資産もあれば、測定される資産もあります。

## AEO 導入メモ

AEO の場合、支払いを試みる前にプロトコル サポートが表示されている必要があります。

次のような機械可読ページまたはエンドポイントを追加します。

1. サポートされている支払いプロトコル
2. 受け入れられるコインまたはレール
3. アップロードの最小サイズと最大サイズ
4. 返金または取り消しポリシー
5. 認証要件
6. エラーコード
7. 冪等性ルール
8. 連絡または紛争の経路

これにより、支払いが非表示の支払いイベントからエージェントが読み取り可能な機能に変換されます。

## よくある質問

**MPP は x402 を置き換えますか?**直接ではありません。 MPP はセッションや継続的な使用に最適です。 x402 は、単純な要求レベルの決済に関してはクリーンなままです。

**ACP は ChatGPT 専用ですか?**
ACP はエージェント コマース用に OpenAI と Stripe を使用して構築されましたが、このプロトコルは ChatGPT 専用の機能ではなく、オープン コマース標準として位置付けられています。

**販売者は x402 と ACP を併用できますか?**
はい。販売者は ACP を使用して支払いを行い、x402 を有料 API アクセス、プレミアム データ、またはエージェント サービスに使用できます。

**ステーブルコインによるマイクロペイメントにはどのプロトコルが最適ですか?**
x402 は、HTTP を介した軽量のステーブルコインのマイクロペイメントにとって最も明確な選択肢です。 Stripe Machine Payments は、Stripe インフラストラクチャ上での暗号決済による自動支払いもサポートしています。
