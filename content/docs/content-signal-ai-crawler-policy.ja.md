#AI トラッカー ポリシーのコンテンツ シグナル

Content-Signal は、Web サイトが自動システムにコンテンツをどのように使用してほしいかを表現する機械可読な方法です。 2026 年 7 月、Cloudflare は robots.txt 内のコンテンツ信号の「use」拡張機能のテストを開始し、信号を AEO、AI 検索、およびエージェントのアクセス ポリシーとより関連性の高いものにしました。

## Content-Sign は何をしますか?

Robots.txt は、クローラーが何を取得できるかを伝えます。 Content-Signal は、取得したコンテンツをどのように使用できるかを表現しようとします。 AI システムはコンテンツをさまざまな方法で読み取り、インデックス付け、要約、再生、トレーニング、または操作できるため、この違いは重要です。

2026 年 7 月 1 日の Cloudflare アップデートでは、提案されたコンテンツ使用設定が 3 つのレベルで追加されます。

|信号値 |単純な意味 |実用化 |
|
---|
---|
---|
| `使用=即時` |保存または再利用せずに対話する |ブラウザ エージェントとタスクの実行 |
| `使用=参照` |インデックス、抽出、リンク | AI 検索と回答の見積もり |
| `use=full` |より完全に要約または再現する |再利用に対する許容度の向上 |

Cloudflareによると、管理対象のrobots.txtファイルに次のような行を含めることができるようになりました。

```text
Content-Signal: search=yes,ai-train=no,use=reference
```

## AEO チームが注意すべき理由

AEO は、AI システムがページをクロールできるかどうかだけを問うものではありません。システムが次のことができるかどうかが重要です。

- ページを発見する
- エンティティまたはタスクを理解する
- ページを正確に引用または参照します
- 許可された制限内でページを使用する
- 必要に応じて安全な行動を完了する

Content-Signal は、[AI Crawlers and robots.txt](/es/docs/ai-crawlers-robots-txt/) と [実行層](/es/docs/execution-layer/) の間にあります。これは、API、認証、価格設定、または同意に代わるものではありません。機械に優先的に軽いコーティングを与えます。

## コンテンツ信号対 robots.txt 対 llms.txt

|ファイルまたは署名 |主な目的 | AEO機能 |
|
---|
---|
---|
| `ロボット.txt` |追跡権限の設定 |基本的なトラッカーへのアクセスを制御する |
|コンテンツ信号 |コンテンツの使用設定 |個別の検索、トレーニング、再利用の意図 |
| `llms.txt` | AI システム向け人間が厳選したナビゲーション |エージェントが適切なページを見つけられるように支援する |
| API ドキュメント |アクションコントラクト |実行と検証をサポート |
| MCPサーバー |ツールインターフェース |エージェントが構造化されたツールを通じて行動できるようにする |

エージェント対応 Web サイトの場合、これらのレイヤーが一致する必要があります。サイトでは、ファイル内のすべての AI トラッカーを許可し、ファイアウォール ルールでブロックし、説明なしに他の場所で MCP ツールを使用するようエージェントに依頼するべきではありません。

## ポリシーパターンの例

|ビジネスモデル |考えられるコンテンツ信号の状態 |なぜ |
|
---|
---|
---|
| SaaS ドキュメント | `検索 = はい、ai-train = いいえ、使用 = 参照` |証拠開示と召喚状は重要です。 || トレーニングを使用することはできません。 Eコマースカタログ | `検索 = はい、使用 = 参照` |製品の可視性と紹介パスが重要 |
|有料エディター |無料ページの `search=yes,ai-train=no,use=reference` |プレミアム コンテンツを保護しながらスニペットを許可する |
|データAPI |公開ドキュメントはオープン、データエンドポイントはクローズ | Content-Signal は支払いを必要としません |
|内部ポータル |追跡を許可しない | AI 発見値は公開されていません |

機械可読サーフェスを変更した後は、[AEO Readiness Checker](/tools/aeo-readiness-checker.html) を使用してください。## 重要な制限事項

Content-Signal は優先シグナルであり、普遍的に強制できるメカニズムではありません。ロボットはそれを無視できます。 Cloudflare ルール、認証、署名されたリクエスト、レート制限、支払いプロトコルなどのインフラストラクチャが引き続き必要になる場合があります。

商用アクセスについては、[エージェント支払いプロトコルの比較](/es/docs/agent-payment-protocols-compared/) および [x402 エージェント支払い](/es/docs/x402-agent-payments/) を参照してください。

## 導入チェックリスト

1. 検索および参照に使用できるコンテンツを決定します。
2. トレーニングの使用を許可するか、制限するか、承認するか、ブロックするかを決定します。
3. robots.txt ルールを追加または確認します。
4. 実際のビジネス ポリシーを反映する場合にのみ、Content-Signal を追加します。
5. 「llms.txt」をエージェントに見つけてもらいたいページに合わせて配置します。
6. クローラーのログと AI 参照を監視します。
7. Cloudflare、Google、Bing、または OpenAI からの主要なトラッカー更新後にポリシーを確認します。

## よくある質問

### Content-Signal はランキング要素ですか?

それが Google のランキング要素であるという証拠はありません。これは、SEO のショートカットではなく、機械可読コンテンツを使用するための優先事項として扱います。

### Content-Signal は llms.txt を置き換えますか?

いいえ、Content-Signal は使用方法の好みを表します。 「llms.txt」は、AI システムが重要なページを見つけるのに役立ちます。

### すべての Web サイトで「use=reference」を使用する必要がありますか?

自動的ではありません。引用や参照が必要な公開コンテンツではこれが合理的なデフォルトですが、有料コンテンツや非公開コンテンツにはより厳格なポリシーが必要です。

### Content-Signal は AI トレーニングをブロックできますか?

「ai-train=no」を表現することもできますが、アプリケーションはトラッカーとインフラストラクチャ制御の動作に依存します。
