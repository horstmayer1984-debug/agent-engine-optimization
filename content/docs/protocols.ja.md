# エージェント エンジン最適化のための AI エージェント プロトコル

AI エージェント プロトコルは、自律システムが機能を発見し、コンテキストを交換し、アクションを要求し、支払いを処理し、結果を検証する方法を定義します。エージェント エンジン最適化の場合、プロトコルは読み取り可能なコンテンツを実行可能なインフラストラクチャに接続します。

## コアプロトコル

|プロトコル | AEOにおける役割 |
|
---|
---|
| PCM |ツールとコンテキスト接続レイヤー |
| A2A / エージェントカード |エージェントの発見とコラボレーション |
| x402 |ネイティブ マシンの支払いと支払い API アクセス |
| UCP |ガバナンス、権限、コントロール プレーン ロジック |
|オープンAPI | API の文書化されたアクション コントラクト |

## どこから始めるべきか

[llms.txt](/llms.txt) などの読み取り可能な検出ファイルから始めて、OpenAPI を使用して API を文書化し、安定したエンドポイントを通じて価値の高いアクションを公開します。

次をお読みください: [MCP 対エージェント API](/es/docs/mcp-vs-api-for-agents/)、[エージェント カードと A2A プロトコル](/es/docs/agent-cards-a2a-protocol/)、[x402 エージェント支払い](/es/docs/x402-agent-payments/)、および [実行層](/es/docs/execution-layer/)。

## 責任に応じてプロトコルを選択する

エージェントの作業全体で 1 つのプロトコルを選択しないでください。各責任が持つ最小のインターフェイスを選択します。

|責任 |適切な出発点 |
|
---|
---|
|既存の HTTP API について説明する |オープンAPI |
|コンテキストに応じたツールまたはリソースを公開する | PCM |
|独立系エージェントの発見と調整 | A2A およびエージェント カード |
|取引機能を公開し、フローを交渉する | UCP |
|機械によって開始された HTTP 支払いを受け入れる | x402 |

境界は重要です。 OpenAPI は、エージェント間のコラボレーションを定義せずにエンドポイントを記述することができます。 MCP は、支払いゲートウェイにならずにツールを公開できます。したがって、プロトコルの決定は、必要な状態変更、信頼制限、および検証方法から開始し、その後に現在の公式仕様をレビューする必要があります。

## 導入前にプロトコルの調整を検証する

既存のインターフェイスと不足している機能に注目してください。安定した REST API がすでにそのアクションをサポートしている場合は、OpenAPI のドキュメントを改善し、バグを修正することで当面の問題が解決される可能性があります。別のプロトコルを追加すると、ユーザーのタスクが改善されずに運用負荷が増加する可能性があります。

検討中のプロトコルについては、そのバージョン、ガバナンス所有者、認証モデル、トランスポート、検出メカニズム、および互換性ポリシーを特定します。次に、非実稼働環境で限定されたワークフローをテストします。テストには、無効なリクエスト、期限切れまたは欠落している認証、新しい試行、および検証可能な最終状態を含める必要があります。プロトコルのサポートは、保守された製品表面です。公開する前に、仕様変更、セキュリティ更新、クライアント互換性、非推奨通知の所有者を割り当てます。

プロトコルのバージョンを各テスト結果および統合ドキュメントとともに保存します。以前のリビジョンを使用していたお客様は、現場、輸送、または認証要件が変更された後に失敗する可能性があります。バージョン管理された証拠により、その欠陥が診断可能になり、チームが同じプロトコル名を持つすべての実装を互換性があるかのように扱うことができなくなります。

宣言されたプロトコル リビジョンまたは認証ポリシーを変更するたびに、互換性を再テストします。

## よくある質問**ほとんどの Web サイトではどのプロトコルを最初に実装する必要がありますか?**  
ほとんどの Web サイトでは、より重いエージェント プロトコルを追加する前に、構造化データ、llms.txt、および OpenAPI ドキュメントから始める必要があります。

**AEO には MCP が必要ですか?**  
いいえ、MCP はツールにアクセスするのに便利ですが、AEO にはコンテンツ、スキーマ、API、信頼シグナル、トランザクション フローも含まれています。

**なぜ x402 が重要ですか?**  
x402 は、従来の人による支払いフローを使用せずに、API 呼び出し、データ アクセス、またはトランザクションの支払い方法をエージェントに提供します。
