#mem0 エージェントのメモリ

エージェントはステートレスな応答から永続的なコンテキストに移行しているため、mem0 は AEO にとって重要です。エージェントが好み、制限、過去の行動、政治的決定を覚えている場合、Web サイトにはより明確な同意、アイデンティティ、パーソナライゼーション、および検証ルールが必要です。

## mem0 が関連する理由

GitHub プラグインは [mem0ai/mem0](https://github.com/mem0ai/mem0) として登場し、2026 年 6 月 28 日にチェックされた GitHub API メタデータには 59,000 を超えるスターが示されていました。リポジトリは、AI エージェントのユニバーサル メモリ層として位置付けられます。

AEO にとって重要な問題は、どのメモリ システムが勝つかということではありません。これは、Web サイトの永続的なエージェント メモリを変更するものです。

[AEO のコンテキスト エンジニアリング](/es/docs/context-engineering-aeo/) および [エージェント フィードバック ループ](/es/docs/agent-feedback-loops/) と併せて読んでください。

##記憶を変えるエージェントは何ですか?

|メモリの種類 |ウェブサイトへの関与 |
|
---|
---|
|ユーザー設定 |製品、出張、およびサポートのフローは、保存された制約を受け入れる必要があります。 |
|過去の行動 |ステータス ページと領収書がより重要になります。 |
|政治的決定 |エージェントには、許可されたアクションとブロックされたアクションの安定した記録が必要です。 |
|アイデンティティリンク |同意とアカウントのリンクが UX の中核要素になります。 |
|エラー履歴 |サポート フローでは、以前の失敗と解決策を公開する必要があります。 |

永続的なメモリはタスクの完了を向上させる可能性がありますが、ガバナンスの問題も引き起こします。

## メモリ対応エージェントの AEO 要件

1. エージェントが記憶できる内容をユーザーが制御できるようにします。
2. 公的な事実を特定のアカウントのデータから分離します。
3. ポリシーと同意のページを簡単に取得できるようにします。
4. 進行中のタスクのステータス エンドポイントを提供します。
5. エージェントが記憶した情報に基づいて行動したときを記録します。
6. ユーザーが古い記憶や間違った記憶を修正できるようにします。
7. 重要な決定については、隠れたカスタマイズに依存しないようにします。

コマースについては、[エージェント ウォレットと支出ガバナンス](/es/docs/agentic-wallets-spend-governance/) および [コマースのエージェント認証](/es/docs/agentic-authentication-commerce/) を参照してください。

## 測定への影響

|メトリック |なぜそれが重要なのか |
|
---|
---|
|繰り返しタスクの成功 |メモリにより、繰り返しの構成作業が軽減されます。 |
|補正率 |ユーザーは誤った仮定を修正できる必要があります。 |
|同意の承諾 |メモリ関数には明示的な信頼が必要です。 |
|ポリシー違反率 |記憶はルールを回避してはなりません。 |
|サポートたわみ |記憶力が向上すると、チケットの繰り返しを減らすことができます。 |

## メモリをトランザクション権限から分離する 記憶された設定はエージェントがタスクを準備するのに役立ちますが、メモリは結果として生じるアクションを黙って許可すべきではありません。区別をワークフローに保存します。 「通常は返金可能な料金を選択する」などの好みが候補リストに影響を与える可能性があります。購入を承認したり、見積もりを無効にしたり、変更された条件を受け入れたりしてはなりません。

|データ型 |安全なデフォルト |
|
---|
---|
|好み |表示オプションを使用して再利用して変更します。
|機密性の高いプロファイル データ |最小限に抑え、保護し、明確な目的を要求する |
|取引制限 |アクションが重大な影響を与える場合を再確認する |
|認可 |特定のアクションと有効期間へのリンク |記憶されたデータを検査、修正、削除する方法を提供します。メモリが現在の命令と競合する場合、現在の命令が優先され、競合は推測されるのではなく記録されるべきです。

## よくある質問

### エージェントの記憶は SEO に効果的ですか?

直接ではありません。記憶力のあるエージェントは、初めて訪問したエージェントとは異なる選択、比較、行動を行う可能性があるため、AEO に影響します。

### 最大のリスクは何ですか?

最大のリスクは、コンテキストが古くなったり、許可されていないことです。 Web サイトでは、ユーザーが記憶に基づいて仮定を検査、修正、取り消しできるようにする必要があります。

### Web サイト自体にエージェント メモリを保存する必要がありますか?

明確なユーザー利益と強力なガバナンスがある場合に限ります。多くのサイトでは、メモリ システムを作成する前に、同意、登録、ステータス ページから始める必要があります。

### これは電子商取引にどのような影響を与えますか?

エージェントはサイズ、予算、配送設定、返品制限、支払い制限を記憶できます。販売者には、エージェントが従うことができるほど明確なポリシーが必要です。
