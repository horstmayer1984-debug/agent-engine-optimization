# エージェントのポートフォリオと支出ガバナンス

エージェント ウォレットを使用すると、AI エージェントはユーザーまたは組織が設定したルール内で支払いを行うことができます。支出ガバナンスは、予算、取引制限、トークンの範囲、監査ログ、コンプライアンスチェック、失効などの制御層です。これらの管理がなければ、エージェントによる支払いは日常的な商取引にとってリスクが高すぎます。

## ウォレットが変化する理由

人間のウォレットは、購入ごとに人が承認することを前提としています。エージェント ウォレットとは、個人または企業がソフトウェアに限定的な権限を委任することを意味します。

これにより、次のような新たな設計上の問題が生じます。

- エージェントには、タスクを完了するのに十分な権限が必要です。
- ユーザーは、エージェントが過剰な支出をしないという確信を必要とします。
- 販売者は、代理店が購入の許可を持っていることを証明する必要があります。
- 発行者、PSP、またはウォレットプロバイダーには監査証跡が必要です。

Fireblocks の Agentic Payments Suite と Mastercard Agent Pay は、この変更を指摘しています。

## 支出ガバナンス管理

|コントロール |例 |
|
---|
---|
|金額制限 |エージェントは購入ごとに最大 50 ドルを支払うことができます |
|商業分野 |エージェントは承認されたサプライヤーからのみ購入できます |
|カテゴリの範囲 |エージェントは事務用品を購入できますが、電子機器は購入できません |
|時間枠 |権限は 7 日後に期限切れになります |
|目的の範囲 |エージェントは SKU ファミリを補充できます |
|承認のしきい値 |限界を超えると人間によるレビューが必要 |
|失効 |ユーザーはエージェント トークンまたはウォレットの付与を無効にできます。
|監査証跡 |各アクションは、意図、支払い、結果を記録します。

これらのコントロールは [実行層](/es/docs/execution-layer/) に不可欠です。

## カードトークンとステーブルコインウォレットの比較

|モデル |強さ |制限 |
|
---|
---|
---|
|カードネットワークのトークン化 |ファミリービジネスの承認、発行者の管理、紛争規則 |認証サインとエージェントの ID が必要 |
|ステーブルコインウォレット |プログラマブル決済とネイティブマシンフロー |コンプライアンス、保管、調整の管理が必要 |
|アカウントクレジット | SaaS と API に便利 | 1 つのプラットフォームに限定 |
|リアルタイム銀行振込・鉄道 |請求書の支払いが迅速な市場に最適 |在庫状況と返金ロジックは異なります |

[エージェント支払いプロトコルの比較](/es/docs/agent-payment-protocols-compared/) はプロトコル層をマッピングします。このページではウォレット制御層に焦点を当てます。

## ビジネスへの影響

販売者は、次の場合により多くのトランザクションを受け取ることを期待する必要があります。

- エージェントはユーザーに代わって行動します
- 支払い手段には適用範囲があります
- 申請書には意図の証拠が含まれています
- ポリシー制限は認可に影響します
- 紛争は代理人の記録に依存します

つまり、決済システムは通常のカード認証よりも多くのコンテキストを保存する必要があります。

## 今準備すべきもの

1. エージェント ID および認可参照用のフィールドを追加します。2.カート、チェックアウト、フルフィルメントのイベントを一緒に記録します。
3. 委任されたエージェントがどの製品を購入できるかを決定します。
4. 人間による確認が必要な高リスクのカテゴリを定義します。
5. エージェントが提供する取引に関する不正行為のルールを確認します。
6. 払い戻しおよびキャンセル明細を機械で読み取り可能な状態に保ちます。

記事 [Fireblocks Agentic Payments Suite](/es/docs/fireblocks-agentic-payments-suite/) では、インフラストラクチャの例について説明しています。

## よくある質問

### エージェントウォレットは通常のデジタルウォレットですか?いいえ。これは、AI エージェントが明示的な制限内で動作できる、委任されたウォレットまたは支払い設定です。

### エージェントのウォレットではカードを使用できますか?

はい。 Mastercard では、特定のカード ネットワーク支払いに対するエージェント トークンとエージェント ペイについて説明しています。 AP2 には、カードと x402 のサンプルも含まれています。

### エージェントウォレットはステーブルコインを使用できますか?

はい。 Fireblocks は、定義された制限および監査証跡内でのステーブルコイン支払いのためのエージェント ウォレットについて説明します。

### 最大のリスクは何ですか?

無制限の権限。エージェントウォレットには、厳格な支出制限、目的の範囲、取り消し、および信頼できるログが必要です。
