2026 年のエージェント エンジン最適化インフラストラクチャ層は 3 つのプロトコルが支配的です。それぞれが異なる問題を解決します。間違ったものを選択すると、開発時間が無駄になります。適切な組み合わせを選択すると、エージェントはデータ アクセスからトランザクション完了までの完全なパスを得ることができます。

MCP (モデル コンテキスト プロトコル) はエー​​ジェントをデータに接続します。 UCP (Universal Commerce Protocol) はコマースのライフサイクル全体を処理します。 ACP (Agent Commerce Protocol) により、購入者がサイトにアクセスしなくても、ChatGPT 内で直接支払いが可能になります。

## MCP: エージェントにデータとツールへのアクセスを許可する

Anthropic は MCP を開発し、Linux Foundation に移植しました。これは垂直統合プロトコルです。 MCP サーバーは、標準化された JSON-RPC インターフェイスを通じて、内部データ ソース、ドキュメント、ツールを AI エージェントに公開します。

MCP は、「このシステムは何を教えてくれるのか、またどのようなツールを使用できるのか」という質問に答えます。これにより、エージェントは CMS にクエリを実行し、ドキュメントを取得し、データベース レコードを確認し、内部関数を呼び出すことができます。支払い、支払いフロー、ビジネス取引を独自に処理することはありません。

エージェントが企業データ (製品カタログ、ドキュメント、ナレッジ ベース) にアクセスする必要がある場合、カスタム統合を行わずに MCP 対応エージェントがツールを検出して使用できるようにしたい場合、および対話がトランザクションではなく読み取りまたはクエリである場合に、MCP を使用します。

技術的な実装は、ツール (エージェントが呼び出すことができる関数)、リソース (エージェントが読み取ることができるデータ)、およびメッセージ (エージェントが使用できるテンプレート) をアドバタイズする JSON-RPC 2.0 サーバーです。 [MCP vs API ガイド] (/docs/mcp-vs-api-for-agents/) では、実装について詳しく説明しています。

## UCP: コマースのライフサイクル全体を管理する

Google は、Shopify、Target、Walmart、Etsy、Wayfair と提携して UCP を開発しました。これは、2026 年 1 月に NRF で開始されました。UCP は、在庫確認、価格交渉、支払い承認、注文の発送、フルフィルメント追跡などのコマース ライフサイクル全体のプロトコルです。

最も重要な技術革新は機能ステートメントです。ストアは、サポートするトランザクション、割引、支払い方法、ポリシーをエージェントに正確に伝える JSON マニフェスト ファイルを /.well-known/ucp に公開します。エージェントは対話を試みる前にこのマニフェストを読み取るため、推測する必要はありません。

UCP は、暗号化された支払い義務と監査証跡のために Agent Payments Protocol (AP2) と連携します。これらを組み合わせることで、エージェントは単一の標準化されたフローを通じて在庫の確認、価格の検証、支払いの承認、注文の送信、履行の追跡を行うことができます。ストアを通じて製品やサービスを販売する場合、Google AI モードまたは Gemini で動作するエージェントに取引を依頼する場合、および発見から履行の確認までの購入ライフサイクル全体が必要な場合に、UCP を使用します。

[Agent Trade Execution の記事] (/docs/agentic-commerce-execution/) では、UCP が電子商取引アーキテクチャでどのように機能するかについて説明しています。

## ACP: ChatGPT 内での支払い

OpenAI は、Stripe と提携して ACP を開発しました。これは、ChatGPT インターフェイス内で直接支払いを処理することで摩擦を最小限に抑えるように特別に設計されたオープンソース プロトコルです。主要なメカニズムは SharedPaymentToken です。購入者の支払い資格情報は、購入者が ChatGPT を離れることなく販売者と安全に共有されます。エージェントは製品の選択を処理し、ACP プロトコルは安全な支払いトークンの交換を処理し、トランザクションは会話内でネイティブに完了します。

ACP は、主要なエージェント主導の販売チャネルが ChatGPT である場合、リダイレクトなしで支払いを行う場合 (購入者が Web サイトにアクセスしない場合)、および Stripe を支払いプロセッサとして使用する場合 (ACP は Stripe インフラストラクチャとネイティブに統合されます) に使用します。

## 3 つのプロトコルがどのように連携するか

これらのプロトコルは競合するものではありません。同じスタックの異なる層が機能します。

MCP はデータ層を提供します。エージェントは MCP を使用して製品カタログにアクセスし、仕様を確認し、ドキュメントを取得します。

UCP はビジネス層を提供します。エージェントが必要なデータを取得すると、UCP は在庫の予約、価格の検証、支払いの承認、注文の発送などのトランザクションを処理します。

ACP は、ChatGPT 専用のインターフェイス層を提供します。安全な支払いトークンを使用して、ChatGPT 会話内で支払いエクスペリエンスをラップします。

完全なエージェント対応インフラストラクチャでは、データ アクセス用の MCP、Google ベースのエージェント ワークフローのビジネス ライフサイクル用の UCP、ネイティブ ChatGPT トランザクション用の ACP の 3 つすべてを使用できます。

## 戦略的意思決定の枠組み

エージェントがデータやツールにアクセスできるようにする場合は、まず MCP を実装します。これは実装が最も速く、幅広いエージェントにサービスを提供します。

製品やサービスを販売し、Google エコシステム エージェントにそれらを購入してもらいたい場合: /.well-known/ucp に機能宣言を含む UCP を追加します。

ChatGPT がビジネスにとって重要な販売チャネルの場合は、Stripe 統合を備えた ACP を追加してください。

最大限のカバレッジが必要な場合は、3 つすべてを実装してください。データ層の MCP、Google エコシステム取引の UCP、ChatGPT 取引の ACP。基礎となるビジネス ロジックは共有されます。プロトコル インターフェイスのみが異なります。

[x402 プロトコルの記事] (/docs/x402-agent-payments/) では、3 つのプロトコルすべてと並行して機能する補完的なマイクロペイメント層について説明します。

## よくある質問

**3 つのプロトコルはすべて必要ですか?**
いいえ。データにアクセスするには MCP から始めてください。どの AI プラットフォームが売上を促進するかに応じて、UCP または ACP を追加します。ほとんどの企業は 1 つの取引プロトコルから開始し、チャネルがそれを保証する場合に 2 つ目の取引プロトコルを追加します。**UCP と ACP は同じサイトに共存できますか?**
はい。これらはエージェントのさまざまなエコシステムにサービスを提供します。 UCP は Google を利用したエージェントを処理します。 ACP は ChatGPT を処理します。同じ製品データとビジネス ロジックは、両方とも異なるプロトコル インターフェイスを通じて機能します。

**MCP は技術製品専用ですか?**
いいえ。エージェントがアクセスする必要があるデータを持っている企業は、MCP の恩恵を受けられます。メニューのあるレストラン、空室状況のあるホテル、サービスの説明のあるアドバイザー。 MCP により、エージェントはあらゆる構造化情報にアクセスできるようになります。

**2026 年 4 月に最も多く採用されているプロトコルはどれですか?**
MCP は最も広く採用されています (数十万のサイト)。 UCP は、小売店でのサポートが充実している主要な取引プロトコルです。 ACP は新しいですが、ChatGPT エコシステム内で急速に成長しています。

**能力宣言とは何ですか?**
/.well-known/ucp に公開される JSON マニフェスト。ストアがサポートしている内容 (利用可能な取引タイプ、支払い方法、割引ルール、ポリシーなど) をエージェントに正確に伝えます。エージェントは対話を試みる前にそれを読みます。
