# Google Genkit エージェント API

Google の Genkit Agents API は、ツール呼び出し、状態、ストリーミング、永続性、インターフェイス プロトコル、人間による承認など、実稼働エージェント アプリケーションがどのように構成されているかを示すため、AEO にとって重要です。 Web サイト チームは、これを、エージェントが単に読みやすいページだけでなく、明示的なタスク サーフェイスを必要としているという信号として扱う必要があります。

## Google が発表した内容

2026 年 7 月 1 日、Google は [Genkit を使用したフルスタック エージェント アプリの構築](https://developers.googleblog.com/build-agentic-full-stack-apps-with-genkit/) を公開しました。 Google は Genkit を、TypeScript、Go、Dart、Python などの言語でエージェントおよび AI ベースのアプリケーションを構築するためのオープンソース フレームワークであると説明しています。

新しいエージェント API は、TypeScript と Go でプレビューとしてマークされています。そのステータスが重要です。これはアーキテクチャ パターンを学習するのに役立ちますが、運用チームは API の変更を予期する必要があります。

## AEO レッスン

Genkit は、一般的なエージェント要件を開発フレームワークにパッケージ化します。

|ゲンキットパターン |ウェブサイトの準備への影響 |
|
---|
---|
|ツールループ |ウェブサイトには明確なアクションと安定した貢献が必要です |
|メッセージ履歴 |エージェントは分離されたページの読み取りではなく、タスクの状態を必要とします。
|トランスミッション |ユーザーは、長時間にわたるエージェントのタスク中に進捗状況を確認する必要があります。
|永続性 |エージェントのワークフローには再開パスと監査パスが必要 |
|人間の承認 |危険な行為には確認ドアが必要 |
|サブエージェント |複雑なタスクには専門のエージェントが関与する場合があります。

これにより、[レスポンス エンジンの最適化](/es/docs/answer-engine-optimization/) と [エージェント エンジンの最適化](/es/docs/what-is-aeo/) の区別が強化されます。回答エンジンはコンテンツを引用します。エージェントも作業を完了する必要があります。

## Web サイトに何を表示するか

エージェント対応サイトでは、重要なタスクを構造化されたワークフローとして公開する必要があります。

1. 適切な製品、サービス、アイテム、またはドキュメントを見つけます。
2. オプションを現在の事実と比較します。
3. 資格、価格、在庫、在庫状況を確認します。
4. 見積もりを依頼し、予約を入れ、または支払いを開始します。
5. 取り消し不能な行動を起こす前に人間による確認を求めます。
6. 成功、失敗、または次のステップの明確なステータスを返します。

それは実際には [実行層](/es/docs/execution-layer/) です。

## Genkit と通常のページの最適化

|要件 | SEOページ |エージェント対応のタスク サーフェス |
|
---|
---|
---|
|主な目的 |ランク付けとクリック獲得 |ユーザータスクを完了する |
|エントリー |チェックを入れて | をクリックします。ユーザーの意図とツールのパラメーター |
|出力 |ページビュー |検証された結果または次のステップ |
|故障モード |低ランクまたはバウンス |間違ったアクション、スタックしたタスク、監査証跡なし |
|測定 |インプレッション、クリック、インタラクション |ツールの呼び出し、承認、完了、エラー |

どちらの層も重要です。ページを最適化するだけのサイトが引用される可能性がありますが、エージェントがアクションを実行しようとすると失敗します。## 実装チェックリスト

1. 上位 5 つのユーザー タスクをマッピングします。
2. 各タスクを文書化したワークフローに変換します。
3. 必須の入力、オプションの入力、およびエラーを定義します。
4. 危険な行為の前に人間の承認を追加します。
5. エージェントのアクションを人間のクリックとは別に記録します。
6. ステータスページと確認ページを機械で読めるようにします。
7. ブラウザ自動化と MCP スタイルのツールを使用してタスクをテストします。より詳細なチェックリストについては、[エージェント向け Web アプリケーションの準備方法](/es/docs/agent-ready-web-apps/) および [エージェント向けの MCP と API](/es/docs/mcp-vs-api-for-agents/) を使用してください。

## よくある質問

### AEO には Genkit が必要ですか?

いいえ、AEO はフレームワークに依存しません。 Genkit は、主要な開発プラットフォームがエージェントの構築をどのように期待しているかを反映しているため、便利です。

### エージェント API のプロダクションは安定していますか?

Google は、2026 年 7 月の発表で Agents API を TypeScript と Go のプレビューとして説明しているため、チームはインターフェイスを変更される可能性があるものとして扱う必要があります。

### マーケティング担当者は Genkit から何を得る必要がありますか?

エージェント アプリケーションにはコンテンツだけでなく、明確なタスクが必要です。マーケティング ページは、エージェントが安全に使用できるフォーム、カタログ、API、またはワークフローに接続する必要があります。

### これは MCP に代わるものですか?

いいえ、Genkit はアプリケーション フレームワークです。 MCP は、モデルをツールやコンテキストに接続するためのプロトコルです。それらは補完的なものになる可能性があります。
