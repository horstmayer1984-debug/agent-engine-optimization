# Agente UX e AEO SEO

O Agent UX afeta o AEO SEO porque as páginas modernas atendem a três públicos: humanos, rastreadores de pesquisa e agentes de IA. Uma página pode classificar e ainda assim reprovar os agentes se as ações, políticas, formulários ou confirmações forem ambíguas. Uma boa experiência do usuário do agente utiliza HTML semântico, dados estruturados, status claros e aprovação humana para ações arriscadas.

## Por que UX pertence à AEO SEO

O SEO tradicional geralmente trata a UX como velocidade da página, usabilidade móvel e clareza de conteúdo. AEO SEO expande UX para incluir usuários de máquinas.

Um agente de IA precisa entender:

- do que se trata a página
- quais ações são possíveis
- quais restrições se aplicam
- quando a aprovação humana é necessária
- se uma ação foi bem-sucedida ou falhou

A página existente [Agent UX e Human-in-the-Loop Design](/es/docs/agent-ux-human-in-the-loop/) cobre o padrão de interface. Esta página conecta esse trabalho à visibilidade da pesquisa e ao AEO.

## UX humano vs. UX do agente

| Elemento de experiência do usuário | Objetivo humano | Objetivo do Agente |
|
---|
---|
---|
| Hierarquia de títulos | Digitalize a página | Inferir tema e significado da seção |
| Tabelas de produtos | Compare visualmente | Extrair atributos |
| Botões | Clique em uma ação clara | Identifique um estoque estável |
| Formulários | Enviar informações | Entenda os campos obrigatórios |
| Mensagens de erro | Recuperar manualmente | Rota do próximo passo |
| Páginas de confirmação | Tranquilize o usuário | Verifique a conclusão da tarefa |

O guia inicial de SEO do Google enfatiza tornar o conteúdo útil e acessível. O Agent UX estende essa ideia para ações legíveis por máquina.

## HTML semântico é a primeira solução

Use reais:

- cabeçalhos
- listas
- mesas
- botões
- rótulos
- formas
- links
- mensagens de status

Não crie conteúdo crítico, como elementos de estilo `div` ou texto somente de imagem. As práticas de acessibilidade ajudam os agentes porque ambas exigem uma estrutura explícita.

## Design humanizado

Nem todas as tarefas devem ser autônomas. Ações de alto risco devem ser interrompidas para revisão:

- pagamento acima de um limite
- alterações de conta
- reservas com penalidades de cancelamento
- decisões médicas, legais ou financeiras
- ações irreversíveis

Para AEO SEO, isso é importante porque os agentes precisam saber quais ações exigem confirmação. Vincule essas regras à página, ao formulário e a qualquer documentação legível por máquina.

## Lista de verificação de implantação

1. Disponibilize o conteúdo principal no servidor ou em HTML rastreável.
2. Utilizar elementos semânticos de comparação, formas e ações.
3. Rotule as ações em linguagem simples.
4. Mostre restrições próximas à ação.
5. Adicione mensagens estruturadas de erro e sucesso.
6. Use páginas de confirmação com URLs estáveis ​​sempre que possível.
7. Padrões de aprovação de documentos.
8. Adicione caminhos de feedback para falhas do agente. O [guia do desenvolvedor AEO](/es/docs/developers-guide-aeo/) e [sites de programação de agentes de IA](/es/docs/programming-websites-for-ai-agents/) se aprofundam na implementação.

## Perguntas frequentes

### A experiência do usuário do agente melhora as classificações?

Indiretamente. Melhora a clareza, acessibilidade, conteúdo estruturado e confiabilidade das tarefas, oferecendo suporte a SEO, pesquisa de IA e AEO.

### O HTML semântico é suficiente para a experiência do usuário do agente?

É a base. Ações complexas também podem exigir APIs, ferramentas MCP, políticas claras e estados de commit.### Cada ação deve ser autônoma?

Não. As ações de alto risco devem exigir aprovação humana ou autoridade específica.

### Qual é a melhoria mais rápida da experiência do usuário do agente?

Substitua tags de ação vagas, campos ocultos e conteúdo somente de script por HTML semântico claro, restrições visíveis e commits estruturados.
