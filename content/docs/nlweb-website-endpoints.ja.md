# NLWeb Web サイトエンドポイント

NLWeb は、Web サイトに自然言語インターフェイスを追加するためのオープン ソースの Microsoft プロジェクトです。これは、Schema.org や RSS などの既存のサイト データを人間と AI エージェントにサービスを提供できるエンドポイントに変えるため、エージェント対応サイトにとって重要です。これは、コンテンツ、検索、MCP スタイルのツール間の便利な橋渡しとなります。

## NLWeb とは何ですか?

[Microsoft NLWeb リポジトリ](https://github.com/microsoft/NLWeb) では、NLWeb を Web サイトの会話型インターフェイスを作成する方法として説明しています。その README には、MCP をネイティブにサポートし、Schema.org と RSS のような半構造化フォーマットを使用し、オープン プロトコルと実装コードを提供すると記載されています。

そのため、NLWeb は、エージェントが盲目的にページを検索するのではなく、構造化された質問をすることを必要とするサイトに適しています。

NLWeb を魔法の SEO レイヤーとして扱うべきではありません。これは Web サイトのインターフェイス パターンに近く、自然言語アクセスを通じてサイト自身の知識とログを公開する方法です。

## NLWeb がキーワードにとって新たな機会となる理由

ほとんどの Web サイトは依然として次の 3 つの層で考えられています。

1. 人間用の HTML ページ。
2. 検索エンジンのサイトマップ。
3. 開発者向けAPI。

NLWeb は、AI 支援エージェントとユーザーのための自然言語エンドポイントという 4 番目のレイヤーをターゲットとしています。

|レイヤー |メインユーザー |典型的な形式 |
|
---|
---|
---|
|ウェブサイト |人間の訪問者 | HTML |
|サイトマップ |検索トラッカー | XML |
|構造化データ |検索システムと人工知能 | Schema.org JSON-LD |
| NLWeb エンドポイント |タスクについて質問する人間または AI エージェント |サイト データによってサポートされる自然言語インターフェイス |
| MCPツール |構造化されたアクションが必要なエージェント |ツール スキームとエンドポイント |

関連するアーキテクチャについては、[MCP リソース、ツール、メッセージ](/es/docs/mcp-resources-tools-prompts/) および [エージェント用の MCP と API](/es/docs/mcp-vs-api-for-agents/) を参照してください。

## どのサイトが最初に NLWeb を参照する必要がありますか?

NLWeb は、豊富で検索可能なデータを含むサイトにとって最も興味深いものです。

1. 電子商取引カタログ。
2. 旅行の在庫。
3. 不動産のリスト。
4. 求人サイト。
5. ドキュメントポータル。
6. ローカルディレクトリ。
7. ナレッジベースをサポートします。
8. イベントデータベース。

これらのサイトにはすでに構造化された事実が含まれています。 NLWeb は、会話形式でエージェントが読み取り可能な回復へのパスを提供します。

## NLWeb の準備方法

NLWeb をまだ実装していない場合でも、データ層を準備します。

1. すべてのページと構造化データにわたって一貫したエンティティ名を使用します。
2. Schema.org マークアップを表示コンテンツと一致させます。
3. 製品、記事、イベント、リスト、文書のクリーンなフィードを維持します。
4. 重要なエンティティごとに安定した正規 URL を作成します。
5. 文書の更新頻度と可用性のルール。
6. どのデータ ブローカーがクエリできるか、どのデータに認証が必要かを決定します。7.潜在的な質問をソース データ フィールドにマッピングします。

これは、[AI エージェント Web サイト プログラミング](/es/docs/programming-websites-for-ai-agents/) および [エージェント対応 Web アプリケーション](/es/docs/agent-ready-web-apps/) と重複します。

## よくある質問

### NLWeb は公式の Web 標準ですか?

NLWeb は、オープン プロトコルと実装コードを備えたオープン ソースの Microsoft プロジェクトです。これを確立された普遍的な標準ではなく、新たなアプローチとして扱います。

### NLWeb は Schema.org に代わるものですか?

いいえ、NLWeb は Schema.org などのサイトや RSS のような形式のデータに依存しています。優れた構造化データは依然として重要です。

### NLWeb は MCP と同じですか?

いいえ、NLWeb は MCP をサポートできますが、MCP はツールとコンテキスト交換のためのプロトコルです。 NLWeb は、自然言語の Web サイト インターフェイスに重点を置いています。

### すべての Web サイトは今すぐ NLWeb を実装する必要がありますか?

いいえ。ユーザーまたはエージェントがサイト上の構造化データについて詳細な質問をする必要があるサイトから始めます。
