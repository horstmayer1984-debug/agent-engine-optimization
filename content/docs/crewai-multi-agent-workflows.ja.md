# CrewAI マルチエージェント ワークフロー

CrewAI は、エージェントの作業を役割間でどのように分割できるかを示すため、AEO にとって重要です。あるエージェントは調査でき、別のエージェントは比較でき、別のエージェントは書き込みでき、別のエージェントは実行できます。 Web サイトは、単一のエージェントの訪問だけでなく、調整されたエージェントのワークフローに対応できるように準備する必要があります。

## CrewAI が関連シグナルである理由

GitHub プラグインは CrewAI エコシステムを表面化し、2026 年 6 月 28 日にチェックされた GitHub API メタデータには、[crewAIInc/crewAI](https://github.com/crewAIInc/crewAI) に 54,000 個以上のスターが含まれていました。

すべての Web サイトに CrewAI が必要というわけではありません。これは、Web サイトのアーキテクチャで委任された作業を考慮する必要があるほど、マルチエージェント パターンが一般的であることを意味します。

より広範なトピックについては、[AI マルチエージェント管理](/es/docs/managing-multiple-ai-agents/) および [マルチエージェント AEO](/es/docs/multi-agent-aeo/) を参照してください。

## 役割ベースのエージェントが Web サイトに必要なもの

|エージェントの役割 |ウェブサイトの要件 |
|
---|
---|
|調査エージェント |明確な定義、出典、比較表 |
|企画エージェント |段階的なワークフローと資格ルール |
|執行エージェント | API、フォーム、ツール、確認ステータス |
|検証エージェント |領収書、ログ、ステータス ページ、監査証跡 |
|ポリシーエージェント |条件、制限、許可されるアクション、およびエスカレーション ルール |

これらのシグナルが散在していたり​​曖昧だったりすると、エージェントは推測に頼ることになります。

## マルチエージェントワークフローの AEO チェックリスト

1. 各主要エンティティの正規ページを公開します。
2. 価格表、制限事項、機能、およびポリシーを追加します。
3. 検出ページを実行ページから分離します。
4. 人間の承認が必要なタスクを定義します。
5. タスクの開始、引き継ぎ、完了を記録します。
6. サポートと登山道を明確にします。
7. ソースページを最新の状態に保ちます。

これは、[エージェントのビジネス上の紛争の証拠](/es/docs/agentic-commerce-dispute-evidence/) および [エージェントの監視可能性と保護の障壁](/es/docs/agent-observability-guardrails/) につながります。

## 複数のエージェントによるリスク

|リスク |緩和 |
|
---|
---|
|矛盾した指示 |正規のポリシー ソースを公開します。 |
|コンテキストがありません |安定したタスク ID とステータス ページを使用します。 |
|安全でない実行 |範囲の承認と許可が必要です。 |
|アトリビューションのギャップ |エージェントの紹介、ツールの呼び出し、結果を追跡します。 |

## エージェントの役割間の契約を定義する

研究者、分析者、実行者などの役割名では、信頼できるワークフローは作成されません。各転送には、定義された入力、出力、所有者、検証ルール、および失敗パスが必要です。次のエージェントは、前のタスクが完了したかどうかを推測する必要はありません。ビジネス タスクの場合、調査機能はソース URL とタイムスタンプを含む製品候補を返すことができます。比較関数は、ランク付けされたリストと拒否されたオプションおよび理由を返すことができます。実行ロールは、明示的な認可制限のある検証済みの選択のみを受け入れる必要があります。

ロールが不完全なデータ、競合するデータ、または結果を返さない場合に何が起こるかをテストします。安全なワークフローは、停止するか、制限内で再試行するか、人間の介入を要求します。実行機能が、それ自体の想定から欠落しているビジネス条件をカバーすることを決して許可すべきではありません。

## よくある質問

### CrewAI は SEO プラットフォームですか?いいえ。これはマルチエージェント フレームワークです。 SEO との関連性は、役割ベースのエージェントが Web サイトのより明確なコンテンツと実行パスを必要とすることです。

### マルチエージェントのワークフローの例とは何ですか?

電子商取引の購入タスクには、調査エージェント、比較エージェント、保険契約エージェント、およびチェックアウト アシスタントが関与する場合があります。

### Web サイトは最初に何を変更する必要がありますか?

まず、事実、ポリシー、アクション、および検証ページを分離します。エージェントはさまざまな理由で各レイヤーを必要とします。

### マルチエージェント設計ではリスクが増加しますか?

できる。代理店が増えると、譲渡、許可、登録も増えます。セキュリティ障壁と強力な監査証跡がより重要になります。
