# AEO SEO チェックリスト: エージェント対応の検索可視性のための 25 のチェック

AEO SEO チェックリストは、クラシック SEO、AI 応答の可読性、およびエージェント対応の実行の 3 つの層をカバーする必要があります。クロール可能なページと有用なコンテンツから始めて、プル可能な応答、構造化データ、内部リンク、llms.txt、ポリシー、および自律エージェントが理解できるアクション パスを追加します。

## レイヤ 1: SEO の基礎

Google の SEO ドキュメントは、検索エンジンがコンテンツをクロール、インデックス付け、理解、表示できるようにするという基本から始まります。その層を飛ばさないでください。

|相談する |承認条件 |
|
---|
---|
| 1. トレーサビリティ |重要な URL は 200 を返し、ブロックされません。
| 2. インデックス作成可能性 |重要なページにはインデックスがありません |
| 3. カノニカル |各ページには正しい正規 URL があります。
| 4. タイトル |タイトルはユニークで、説明的で、クリックしたくなるものです |
| 5. メタディスクリプション |説明はページの価値を説明します |
| 6.H1 |明確な H1 はページの意図と一致します。
| 7. 内部リンク |重要なページは関連ハブからリンクされています。
| 8. サイトマップ | XML サイトマップにはインデックス可能な正規 URL が含まれています。

完全な監査フローについては、[AEO 準備監査](/es/docs/audit/) を使用してください。

## レイヤ 2: AI 応答の準備

AI 応答システムには取り外し可能な構造が必要です。それはロボットのような文章を書くという意味ではありません。それは、明確な答え、適切なタイトル、情報源によって裏付けられた声明を意味します。

|相談する |承認条件 |
|
---|
---|
| 9. 直接的な対応 |最初の 80 語が主な質問に答えます |
| 10. 質問 H2 |セクションは実際のフォローアップの質問と一致します。
| 11. テーブル |比較と基準が構造化されている |
| 12. 定義 |重要な用語は一度定義され、一貫して定義されます。
| 13. 情報源 |苦情を一次情報源または信頼できる情報源にリンクする |
| 14. よくある質問 | FAQ が実際の検索の質問に答えます |
| 15. エンティティの一貫性 |名前、プロトコル、製品、コンセプトは安定しています |
| 16. コンテンツの鮮度 |ページには現在のプロトコルまたは市場のステータスが表示されます。

[AI 検索最適化フレームワーク](/es/docs/ai-search-optimization/) では、このレイヤーについて詳しく説明しています。

## レイヤ 3: エージェント エンジンの最適化

エージェント エンジンの最適化は、システムが読み取りだけでなく使用できるようになったときに始まります。

|相談する |承認条件 |
|
---|
---|
| 17.llms.txt |エージェント検出のための重要なページがリストされています |
| 18. 機械可読ポリシー |返品、価格、資格、制限は明確です |
| 19. 製品またはサービスのデータ |エージェントは推測せずに属性を比較できます。
| 20. 行動ルート |フォーム、カート、API、または予約ステップは安定しています |
| 21. 認可と許可モデル |リスクの高い行動には特定の承認または権限が必要です。
| 22. エラー状態 |失敗すると、有用なステータスと次のステップが返されます。
| 23. 確認状態 |実行されたアクションは検証可能です。
| 24. 帰属 | AI 紹介とエージェント ソース セッションが測定される |
| 25. フィードバックループ |エージェントの失敗が解決策になる |

[実行レイヤー](/es/docs/execution-layer/) および [AEO 実装ガイド](/es/docs/implement-aeo/) は、このレイヤーをカバーしています。

## チェックリストの使用方法「AEO SEO」、「エージェント エンジンの最適化」、または関連する用語からすでにインプレッションを獲得しているページから始めます。 Search Console のパフォーマンス レポートでは、どのクエリとページが表示されているかを確認できます。次に、新しいページを作成する前に、最も印刷量の多いページを強化します。

この順序が重要です。

1.ランキングページを修正。
2. さまざまなサブトピックについて 1 つまたは 2 つのサポート ページを追加します。
3. リンクします。
4. クエリの移動と CTR を測定します。
5. 新しい検索意図が表示された場合にのみ展開します。

## よくある質問

### このチェックリストは SEO 用ですか、AEO 用ですか?

両方。 AEO には検出が必要であるため、SEO から始まり、次に実行層のチェックと AI 応答が追加されます。

### 各ページは 25 のチェックすべてに合格する必要がありますか?

いいえ。情報ページにはアクション エンドポイントが必要ない場合があります。トランザクション ページと電子商取引ページでは、ランタイム層でさらに多くのチェックが必要です。

### 最速の勝利とは何ですか?

すでにインプレッションはあるものの CTR が低いページ (タイトル、メタディスクリプション、ダイレクトレスポンス、内部リンク、FAQ 構造) を改善します。

### チェックリストはどのくらいの頻度で繰り返す必要がありますか?

Search Console でのコンテンツの大幅な変更、プロトコルの更新、製品の発売、クエリの変更の後に、この作業を繰り返します。
