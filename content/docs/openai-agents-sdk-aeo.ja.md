# OpenAI エージェント SDK と AEO

OpenAI Agent SDK は、ツール、ハンドオフ、ガードレール、追跡、マルチエージェント ワークフローなど、エージェントが実際にどのように作成されるかを示すため、AEO にとって重要です。 Web サイト チームはプロジェクト間で SDK をコピーしないでください。エージェントは、デジタル サービスを操作するときにエージェントが期待するインターフェイス パターンを学習する必要があります。

## このリポジトリが AEO コンテンツ グループに属しているのはなぜですか?

GitHub プラグインは、著名なエージェント フレームワーク リポジトリとして [openai/openai-agents-python](https://github.com/openai/openai-agents-python) として登場しました。 2026 年 6 月 28 日にチェックされた GitHub メタデータには、27,000 を超えるスターとアクティブなデフォルト ブランチが示されていました。

その人気は役に立ちますが、SEO の観点は「SDK とは何ですか?」ということではありません。最も良い角度は、SDK によって作成されたエージェントが Web サイトを安全に使用できるようにするために、Web サイトに何を公開する必要があるかということです。

これは、[実行層](/es/docs/execution-layer/)、[UX および AEO SEO エージェント](/es/docs/agent-ux-aeo-seo/)、[AEO の実装方法](/es/docs/implement-aeo/) に直接接続します。

##エージェント SDK 設計から得た AEO の教訓

| SDK パターン | AEO がウェブサイトに与える影響 |
|
---|
---|
|ツール |明確なアクション、パラメータ、制限、結果を公開します。 |
|転送 |あるエージェント、チーム、またはワークフローが別のエージェント、チーム、またはワークフローに制御を渡すタイミングを定義します。 |
|手すり |購入やアカウントの変更などの危険なアクションの前にポリシーチェックを追加します。 |
|追跡 |意思決定、ツールの呼び出し、エラー、承認を記録します。 |
|マルチエージェントのワークフロー |検出、決定、実行、検証のパスを分離します。 |

これらは開発者だけの懸念ではありません。これらは、エージェントがサイト、API、またはビジネス プロセスを解釈する方法を形成します。

## Web サイト チームが公開すべき内容

エージェント対応の Web サイトでは、重要な操作を明確にする必要があります。

1. 製品カタログを検索します。
2. 価格と在庫状況を確認します。
3. 返品ポリシーをお読みください。
4. 人間の承認を得て支払いを開始します。
5. サポートチケットを作成します。
6. デモをリクエストします。
7. API ドキュメントを入手します。
8. タスクの完了を確認します。

各アクションには、入力、出力、許可された状態、およびエラー処理が定義されている必要があります。ボタンのラベルだけでは十分ではありません。

## 導入チェックリスト

|タスク |なぜそれが重要なのか |
|
---|
---|
|安定したタスク URL を作成する |エージェントには正規のエントリ ポイントが必要です。 |
|ドキュメント API オペレーション |ツールを使用するエージェントには明確なスキーマが必要です。 |
|人間の承認ステータスを追加する |危険なアクションはサイレントに実行すべきではありません。 |
|ツール呼び出しログを記録する |チームはエージェントの動作をデバッグおよび監査する必要があります。 |
|公的文書を常に最新の状態に保つ |エージェントは人間と同じように古い文書に従います。 |

API 指向の作業の場合は、これを [MCP vs Agent API](/es/docs/mcp-vs-api-for-agents/) および [MCP Resources vs Tools vs Prompts](/es/docs/mcp-resources-tools-prompts/) と組み合わせます。

## よくある質問

### AEO には OpenAI Agent SDK が必要ですか? いいえ。 AEO は単一の SDK に関連付けられていません。 SDK は、ツール、状態、ガードレール、追跡、転送などのエージェントの共通アーキテクチャを反映しているため便利です。

### これは通常の SEO に代わるものですか?

いいえ、SEO によってページが発見されます。エージェント対応アーキテクチャは、エージェントがタスクを理解した後に行動を起こすのに役立ちます。

### Web サイトに加える最初の変更は何ですか?最も価値の高いタスクを、入力、出力、エラー、承認ルールを含む構造化されたワークフローとして文書化します。

### マーケティング担当者は SDK を気にする必要がありますか?

はい。サイトが、エージェントがユーザーに代わって操作できるリード フォーム、カート、予約、サポート フロー、または製品比較に依存している場合は可能です。
