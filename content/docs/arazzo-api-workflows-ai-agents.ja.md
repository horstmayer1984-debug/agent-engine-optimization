# AI エージェント向けの Arazzo API ワークフロー

OpenAPI は個々の API 操作を記述しているため、Arazzo は AEO にとって重要ですが、エージェントは多くの場合ワークフローを必要とします。予約、見積もり、支払い、またはオンボーディングのタスクでは、正しい順序で複数の API 呼び出しが必要になる場合があります。 Arazzo は、これらのシーケンスを記述するための機械可読な方法をチームに提供します。

## アラッツォとは

OpenAPI Initiative では、Arazzo を、結果を達成するための API 呼び出しのシーケンスとその依存関係を表現するための仕様として説明しています。 OpenAPI を置き換えるのではなく、OpenAPI を補完します。

一次情報源:

- [OpenAPI イニシアチブ: Arazzo 仕様](https://www.openapis.org/arazzo-specification)
- 【アラッツォ仕様】(https://spec.openapis.org/arazzo/latest.html)
- [OpenAPI仕様](https://spec.openapis.org/oas/v3.1.0.html)

##OpenAPI だけでは不十分な理由

OpenAPI はエンドポイントを記述するのに最適です。これは、開発者またはマシンに、どのような操作が存在するか、どのようなパラメーターを受け入れ、どのような応答を返すかを伝えます。

しかし、実際のタスクの多くには順序が必要です。

1. 利用可能なサービスを検索します。
2. サービスを選択します。
3. 利用可能なスペースを確認します。
4. 予約を作成します。
5. 予約を確認します。

エージェントが個々のエンドポイントのみを認識する場合、シーケンスを推測する必要があります。これは単純な API では機能する可能性がありますが、依存関係、トークン、状態、または検証ルールが関係すると不安定になります。

## OpenAPI 対 Arazzo 対 MCP

|レイヤー |主な仕事 |エージェントの特典 |
|
---|
---|
---|
|オープンAPI | API の操作について説明します |利用可能なエンドポイントについて |
|アラッツォ |オペレーション全体にわたるワークフローについて説明します。タスクを完了する方法を理解する |
| PCM |呼び出し可能なツールとリソースを公開します。エージェント インターフェイスを介してアクションを呼び出す |

[MCP vs API ガイド](/es/docs/mcp-vs-api-for-agents/) では、ツール層について説明しています。 Arazzo は、これらのツールの背後にあるワークフロー ロジックの文書化を支援しています。

## アラッツォが AEO に適合する場所

AEO は、エージェントが次のシーケンスを実行できる場合に最も強力になります。

1. サービスを発見します。
2. 機能を理解する。
3. 読み取り制限。
4. ワークフローを実行します。
5. 結果を確認します。

Arazzo は、複数ステップのワークフローを明示的にすることで、ステップ 2 ～ 5 をサポートします。これは、エージェントがコンテンツをフェッチする以上のことを行う必要がある [実行層](/es/docs/execution-layer/) で特に役立ちます。

## 強力な使用例

|使用例 |アラッツォが役立つ理由 |
|
---|
---|
| Eコマース決済 |カート、配送、税金、支払い、確認が順番に行われます。
| B2B 見積もりリクエスト |資格、構成、価格、配送は相互に依存します |
|旅行の予約 |空席状況、運賃規則、乗客データおよび確認を注文する |
| SaaS オンボーディング |ワークスペース、ユーザー、請求、および権限の設定には依存関係があります ||サポートクライミング |チケットの作成、分類、添付およびルーティングのステータスが必要 |
|財務ワークフロー |行動を起こす前に、開示、適格性、同意が得られる必要があります。

[エージェント対応 Web アプリケーション チェックリスト](/es/docs/agent-ready-web-apps/) では、製品の基礎となる表面を設計する方法について説明します。

## Arazzo ワークフローに含めるべきもの

少なくとも次のことを文書化します。

- ワークフローの目標
- チケットが必要です
- 順序付けられたステップ
- OpenAPI ソースの操作
- ステップ間の依存関係
- 成功基準
- 出発
- エラー状態
- 再試行と冪等性の期待

成功基準は特に重要です。エージェントは「完了」が何を意味するのかを知る必要があります。## ワークフローフォームの例

予約 API の場合:

|ステップ |操作 |後で使用される出力 |
|
---|
---|
---|
| 1 | `リストサービス` | `サービスID` |
| 2 | `利用可能なスロットのリスト` | `スロットID` |
| 3 | `create_booking` | `予約ID` |
| 4 | `get_booking` |確認済みステータス |

ワークフローの文書化がないと、エージェントが最初に間違ったエンドポイントを呼び出したり、確認手順を見逃したりする可能性があります。

## デプロイメントパス

1. OpenAPI ドキュメントをクリーンアップします。
2. 価値の高いワークフローを 3 つ選択します。
3. これらのワークフローの Arazzo 説明を作成します。
4. ワークフロー エージェントまたはランナーを使用してテストします。
5. API ドキュメントと「llms.txt」からワークフロー ドキュメントをリンクします。
6. 必要に応じて、同じワークフローを MCP ツールとしてパッケージ化します。

これにより、システムが安定した状態に保たれます。OpenAPI は操作を記述し、Arazzo はシーケンスを記述し、MCP は呼び出し可能なツールを公開します。

## よくある質問

### Arazzo は OpenAPI を置き換えますか?

いいえ。OpenAPI 操作を使用するワークフローを記述することで OpenAPI を補完します。

### Arazzo は AI エージェント専用ですか?

いいえ。ドキュメント、SDK の生成、テスト、コンプライアンスに役立ちます。 AI エージェントは強力なユースケースです。

### MCP の前にアラッツォは必要ですか?

いつもではありません。単純なツールの場合は、MCP で十分な場合があります。マルチステップのワークフローの場合、Arazzo はプロセスを明示するのに役立ちます。

### SEO の観点とは何ですか?

アラッツォはランキング要素ではない。ワークフローを機械可読にすることで、実行層の準備を向上させます。

### どのワークフローを最初に文書化する必要がありますか?

収益、サポート解決、予約、オンボーディング、またはコンプライアンスに配慮したアクションに関連付けられたワークフローから始めます。
