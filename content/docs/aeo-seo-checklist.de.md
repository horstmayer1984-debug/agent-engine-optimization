# AEO SEO-Checkliste: 25 Checks für agentenbereite Suchsichtbarkeit

Eine AEO-SEO-Checkliste sollte drei Ebenen abdecken: klassisches SEO, Lesbarkeit der KI-Antworten und agentenbereite Ausführung. Beginnen Sie mit crawlbaren Seiten und nützlichen Inhalten und fügen Sie dann abrufbare Antworten, strukturierte Daten, interne Links, llms.txt, Richtlinien und Aktionspfade hinzu, die autonome Agenten verstehen können.

## Ebene 1: SEO-Grundlage

Die SEO-Dokumentation von Google beginnt immer noch mit den Grundlagen: Sie helfen Suchmaschinen beim Crawlen, Indexieren, Verstehen und Anzeigen Ihrer Inhalte. Überspringen Sie diese Ebene nicht.

| Konsultieren | Genehmigungsbedingung |
|
---|
---|
| 1. Rückverfolgbarkeit | Wichtige URLs geben 200 zurück und sind nicht blockiert |
| 2. Indexierbarkeit | Wichtige Seiten haben keinen Index |
| 3. Kanonische Texte | Jede Seite hat eine korrekte kanonische URL |
| 4. Titel | Titel sind einzigartig, beschreibend und anklickbar |
| 5. Meta-Beschreibungen | Beschreibungen erläutern den Wert der Seite |
| 6.H1 | Ein klares H1 entspricht der Absicht der Seite |
| 7. Interne Links | Wichtige Seiten werden von relevanten Hubs aus verlinkt |
| 8. Sitemap | XML-Sitemap enthält indizierbare kanonische URLs |

Für einen vollständigen Prüfungsablauf verwenden Sie [AEO-Vorbereitungsprüfung](/es/docs/audit/).

## Ebene 2: Bereiten Sie sich auf die KI-Reaktion vor

KI-Reaktionssysteme benötigen eine abnehmbare Struktur. Das bedeutet nicht, dass Roboter schreiben. Es bedeutet klare Antworten, gute Titel und durch Quellen untermauerte Aussagen.

| Konsultieren | Genehmigungsbedingung |
|
---|
---|
| 9. Direkte Antwort | Die ersten 80 Wörter beantworten die Hauptfrage |
| 10. Fragen H2 | Abschnitte entsprechen echten Folgefragen |
| 11. Tabellen | Vergleiche und Kriterien sind strukturiert |
| 12. Definitionen | Wichtige Begriffe werden einmalig und einheitlich definiert |
| 13. Quellen | Verknüpfen Sie Beschwerden mit primären oder zuverlässigen Quellen |
| 14. Häufig gestellte Fragen | FAQ beantwortet echte Suchfragen |
| 15. Entitätskohärenz | Namen, Protokolle, Produkte und Konzepte sind stabil |
| 16. Aktualität der Inhalte | Seiten zeigen den aktuellen Protokoll- oder Marktstatus |

Das [AI Search Optimization Framework](/es/docs/ai-search-optimization/) erklärt diese Ebene ausführlicher.

## Schicht 3: Agent Engine-Optimierung

Die Optimierung der Agent Engine beginnt, wenn das System genutzt und nicht nur gelesen werden kann.

| Konsultieren | Genehmigungsbedingung |
|
---|
---|
| 17. llms.txt | Wichtige Seiten zur Agentenerkennung werden aufgelistet |
| 18. Maschinenlesbare Richtlinien | Rendite, Preise, Berechtigung und Limits sind klar |
| 19. Produkt- oder Dienstleistungsdaten | Agenten können Attribute vergleichen, ohne zu raten |
| 20. Aktionsroute | Formulare, Warenkörbe, API oder Buchungsschritte sind stabil |
| 21. Autorisierungs- und Berechtigungsmodell | Maßnahmen mit hohem Risiko bedürfen einer besonderen Genehmigung oder Befugnis |
| 22. Fehlerzustände | Fehler geben nützliche Statusmeldungen und nächste Schritte zurück |
| 23. Bestätigungszustände | Die durchgeführten Aktionen sind nachweisbar |
| 24. Namensnennung | KI-Verweise und Agentenquellensitzungen werden gemessen |
| 25. Rückkopplungsschleife | Aus Versagen der Agenten werden Lösungen |

Die [Ausführungsschicht](/es/docs/execution-layer/) und der [AEO-Implementierungsleitfaden](/es/docs/implement-aeo/) decken diese Schicht ab.

## So verwenden Sie die ChecklisteBeginnen Sie mit Seiten, die bereits Impressionen von „AEO SEO“, „Agent Engine Optimization“ oder verwandten Begriffen erhalten. Der Leistungsbericht der Search Console kann zeigen, welche Suchanfragen und Seiten an Sichtbarkeit gewinnen. Verbessern Sie dann die Seiten mit den meisten Drucken, bevor Sie neue Seiten erstellen.

Diese Reihenfolge ist wichtig:

1. Ranking-Seite korrigieren.
2. Fügen Sie eine oder zwei Supportseiten für verschiedene Unterthemen hinzu.
3. Verlinken Sie sie.
4. Messen Sie die Anfragebewegung und die CTR.
5. Erweitern Sie nur, wenn eine neue Suchabsicht angezeigt wird.

## Häufig gestellte Fragen

### Ist diese Checkliste für SEO oder AEO?

Beide. Es beginnt mit SEO, da AEO eine Erkennung benötigt, und fügt dann Prüfungen auf Ausführungsebene und KI-Reaktion hinzu.

### Sollte jede Seite alle 25 Prüfungen bestehen?

Nein. Informationsseiten benötigen möglicherweise keine Aktionsendpunkte. Transaktions- und E-Commerce-Seiten erfordern mehr Prüfungen auf der Laufzeitebene.

### Was ist der schnellste Sieg?

Verbessern Sie Seiten, die bereits Impressionen, aber eine schwache CTR haben: Titel, Meta-Beschreibung, direkte Antwort, interne Links und FAQ-Struktur.

### Wie oft sollte die Checkliste wiederholt werden?

Wiederholen Sie dies nach größeren Inhaltsänderungen, Protokollaktualisierungen, Produkteinführungen und Abfrageänderungen in der Search Console.
