x402 は、エージェント エンジンの最適化を、発見可能性とアクションから完全に収益化可能な自律型経済に変換します。 HTTP 402 (支払いが必要) を、AI エージェントがステーブルコインで即座に使用できるネイティブ支払いレイヤーにします。アカウントもセッションも人間の承認もありません。

2026 年 4 月まで、AEO は検出 (llms.txt、エージェント カード)、理解 (構造化データ、スキーマ マークアップ)、およびアクション (MCP エンドポイント、API) の 3 つのレイヤーに焦点を当てていました。 x402 は 4 番目の支払いを追加します。エージェントは、1 つのシームレスな HTTP フローでサービスを発見し、理解し、対応し、料金を支払うことができるようになります。

## x402 支払いフローの仕組み

フローには 3 つのステップがあります。

ステップ 1: エージェントは通常の HTTP リクエストをエンドポイントに送信します。ステップ 2: サーバーは、価格、受け入れられたトークン (Base または Solana の USDC)、受益者のウォレット アドレス、支払い条件を含む構造化された JSON 本文を含む 402 応答を返します。ステップ 3: エージェントは支払い証明を含む X-PAYMENT ヘッダーを構築し、リクエストを再試行し、アクセスを受け取ります。

往復には 2 秒もかかりません。取引手数料は 0.001 ドル未満です。アカウント作成、セッション管理、支払いフォームは必要ありません。

エージェントにとって、これは認証を使用することでできるだけ簡単になります。サイト運営者にとって、すべての API 呼び出しまたはコンテンツ要求は自動ログインになります。

## これが AEO アーキテクチャにとって重要な理由

x402 より前は、エージェント トラフィックを収益化するには、従来のサブスクリプション モデルまたは API キー管理が必要でした。どちらも、サインアップし、支払いの詳細を入力し、アカウントを管理する人間がループのどこかにいることを前提としています。

エージェントのトラフィッキングはそのようには機能しません。 MCP を通じてプレミアム データ ソースを発見し、ツールからスキーマを取得し、すぐに電話をかけたいフリーランス エージェントは、アカウントを登録するために一時停止する必要はありません。 x402 ではその一時停止が解消されます。

これにより、エージェント対応サイトのビジネス モデルが変わります。エージェントを使用できる人間にサブスクリプションを販売するのではなく、リクエストごとにアクセスをエージェントに直接販売します。価格設定は細かく設定できます。データ クエリの場合は 0.001 ドル、コンピューティング オペレーションの場合は 0.01 ドル、プレミアム分析の場合は 0.10 ドルです。

## 財団とエコシステム x402

Coinbase は、2026 年 4 月 2 日に x402 を Linux Foundation に移行しました。創設メンバーには、Google、Stripe、AWS、Visa、Mastercard、Shopify、Microsoft が含まれます。オープンな仕様により、特定の企業が支払いレイヤーを制御することはありません。

Base (Coinbase のレイヤー 2 ネットワーク) は、x402 トランザクション量の約 75 パーセントを 1 秒未満のファイナライズとほぼゼロの手数料で処理します。 Solana は 2 番目に活発なチェーンです。Nevermined は、従来の支払いインフラストラクチャを使用する必要があるエージェント向けに、x402 と Visa Rails を組み合わせて、2026 年 4 月 9 日にエージェント カード決済を開始しました。

## 比較: x402 の前後の AEO

|外観 | x402 より前の AEO | x402 で AEO が有効 |
|---|---|---|
|エージェントの対話 |発見して行動する |自律的に発見、行動、支払い |
|収益化 |購読または広告 |リクエストごとのネイティブのマイクロペイメント |
|成功指標 | API コールと予約 |エージェント主導の収益と維持 |
|支払いの摩擦 |アカウントの作成が必要です |ゼロフリクション、単一の HTTP ヘッダー |
|収入の粒度 |毎月または毎年 |リクエストまたはアクションによる |

## サイト運営者にとってこれが意味することMCP エンドポイントとアクション スキーマを備えた [実行層](/es/docs/execution-layer/) がすでにある場合、x402 の追加がエージェント トラフィックを収益に変える最後のステップです。このミドルウェアは軽量です (詳細は [x402 実装ガイド](/es/docs/implementing-x402/) を参照)。

まだ読書層を構築中の場合は、まずそこに焦点を当ててください。 x402 は実行層を収益化します。発見可能で実用的なコンテンツがなければ、収益化できるものは何もありません。

[エージェント ネイティブ ビジネス モデルの記事] (/docs/agent-native-business-models/) では、x402 がアテンション エコノミーからペイパータスク エコノミーへの広範な移行にどのように適合するかを説明しています。

## よくある質問

**x402 とは何ですか?**
HTTP ステータス コード 402 (支払いが必要) を使用して、AI エージェントと Web サービス間の即時マイクロペイメントを可能にするオープン プロトコル。エージェントは、HTTP 経由で直接ステーブルコイン (主に USDC) で支払います。

**x402 を展開するには暗号通貨インフラストラクチャが必要ですか?**
支払いを受け取るにはウォレットアドレスが必要です。 x402 ミドルウェアは支払いの検証を処理します。ブロックチェーンインフラを構築する必要はありません。 Node.js、Python、Cloudflare Workers、およびその他のプラットフォーム用に事前に構築されたミドルウェアがあります。

**リクエストごとにいくら請求できますか?**
いくらでも。一般的な範囲は、単純なデータ クエリの場合は 0.001 ドル、複雑な計算やプレミアム データの場合は数ドルです。各端末の価格は、それが提供する価値に基づいて決定します。

**X402 は AI エージェント専用ですか?**
このプロトコルはあらゆる HTTP クライアントで機能しますが、従来の支払いフローでは摩擦が生じる自律的なマシン間支払い向けに設計されています。人間のユーザーは従来の支払い方法を好む傾向があります。

**エージェントによる支払いが失敗した場合はどうなりますか?**
サーバーは 402 応答を返します。エージェントは、修正された支払いで再試行するか、別のサービスに移動することができます。支払いが失敗した場合はアクセスが許可されません。
