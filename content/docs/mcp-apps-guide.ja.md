# MCP アプリケーション: エージェント ワークフロー用の対話型インターフェイス

MCP アプリケーションは、モデル コンテキスト プロトコルの公式拡張機能であり、ツールがテキストや構造化データだけでなく対話型ユーザー インターフェイスを返すことを可能にします。一部のエージェント ワークフローでは、会話自体内にグラフ、承認フォーム、ダッシュボード、または複数ステップのコントロールが必要なため、これは重要です。 MCP アプリケーションは、ワークフローをコンテキスト内に保ちながら、プレーン テキストでは不十分な場合に、より豊富なインターフェイスをユーザーに提供します。

## MCP の公式文書に記載されていること

MCP プロジェクトは、2026 年 1 月に最初の公式拡張機能として MCP Apps を発表しました。公式ドキュメントでは、MCP ホスト内でレンダリングされるインタラクティブな HTML インターフェイスについて、サンドボックス化とアプリケーションとホスト間の安全な通信ブリッジについて説明しています。

## MCP アプリケーションが存在する理由

多くのツールの回答にはテキストで十分です。

- 検索結果
- 状態
- 簡単な推奨事項

しかし、テキストは次のようにぎこちなくなります。

- グラフィックス
- 視覚的な比較
- 承認ワークフロー
- デザイナーキャンバス
- 複数の依存フィールドを含むフォーム
- ライブステータスを表示するダッシュボード

MCP アプリケーションは、ユーザーをホスト エクスペリエンスから強制的に離れることなく、ユーザー インターフェイスのレイヤーを追加します。

## MCP アプリと通常の Web アプリの比較

|質問 |スタンドアロン Web アプリケーション | MCP アプリケーション |
|
---|
---|
---|
|どこにレンダリングされますか? |サイトまたは別のタブ | MCP ホストの内部 |
|会話のコンテキストは保存されていますか? |一般的にはそうではありません |はい |
|に最適 |完成品 |ワークフロー スニペットとツールにリンクされた UI |
|コミュニケーション |ブラウザ アプリケーション API | MCP プラス アプリケーション ブリッジ |

MCP アプリケーションは、すべての Web アプリケーションを置き換えるものではありません。これらは、インターフェイスが製品全体ではなく、エージェントが仲介するタスクの一部である場合に役立ちます。

## AEO への影響

[実行層](/es/docs/execution-layer/) は多くの場合、人間によるチェックポイントを必要とします。 MCP アプリケーションは、そのチェックポイントを実用的にすることができます。

- 払い戻しを承認する
- 見積書を調べる
- オプションから選択します
- グラフを確認する
- 危険な行為を確認する

これは自然に [エージェントのユーザー エクスペリエンスと人間参加型デザイン](/es/docs/agent-ux-human-in-the-loop/) につながります。そこでは、問題は「人間かエージェントか」ではなく、「どの部分にどのインターフェイスが必要か?」ということになります。

## MCP アプリケーションを使用する場合

次の場合に使用します。

- ツールの結果には視覚的な解釈が必要です
- 次のステップでは構造化された人間の入力が必要です
- 会話のコンテキストを保持したい場合
- ユーザー インターフェイスはツール ワークフローに従属します

次の場合には使用しないでください。

- 単純なテキスト応答で十分です
- エクスペリエンスには完全で独立した製品が必要です
- 対象のホスティング エコシステムは拡張機能をまだサポートしていません## セキュリティと展開

公式ドキュメントには、サンドボックス レンダリングと安全なメッセージ ブリッジについて説明されています。つまり、チームは依然として以下について慎重に検討する必要があります。

- ユーザーインターフェイスに入力されるデータ
- どのツールが UI 呼び出しをトリガーできるか
- 承認が必要な場合
- ステータスの記録方法
- ホストのサポートの違い豊富なユーザー インターフェイスでは厳格な権限の必要性が排除されないため、[MCP 認証ガイド](/es/docs/mcp-authorization-oauth-ai-agents/) は引き続き有効です。

## よくある質問

### MCP アプリケーションはメイン MCP の一部ですか?

これらは公式の拡張機能であり、コア プロトコル自体ではありません。

### MCP アプリケーションは Web サイトに置き換わりますか?

いいえ。これらはオンライン ワークフロー UI に最適であり、すべてのパブリック Web エクスペリエンスを置き換えるものではありません。

### どのクライアントがあなたをサポートしていますか?

ホストのサポートはさまざまです。 MCP ドキュメントでは、公式拡張機能のカスタマー サポート マトリックスが維持されています。

### 最初に最適な使用例は何ですか?

承認フローと小さなダッシュボードは、テキスト以上のものを必要とし、完全な製品のユーザー インターフェイス未満のものを必要とするため、確実な出発点となります。

## 結論

エージェントのワークフローには依然として人間による判断が必要であるため、MCP アプリケーションは重要です。これらは、エージェントがすでに動作しているのと同じ場所に、有用なインターフェイスのフラグメントをもたらします。
