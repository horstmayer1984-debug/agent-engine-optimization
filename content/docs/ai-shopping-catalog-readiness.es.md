# Preparación del catálogo de compras con IA: datos de productos para agentes

La preparación del catálogo de compras con IA significa que los sistemas de IA pueden encontrar, comparar, confiar y agregar sus productos al carrito. Un catálogo legible es más que un esquema de producto. Necesita identificaciones estables, variantes, precio y estado de inventario, reglas de envío, políticas de devolución, imágenes, reseñas y una estructura suficiente para que los agentes eviten adivinar.

## Por qué los catálogos ahora son estratégicos

El comercio agente comienza antes del pago. Un agente no puede comprar lo que no puede interpretar. La investigación de comerciantes de PayPal destaca los datos de productos legibles por máquina como un paso práctico de preparación, mientras que Google describe Universal Cart y UCP como infraestructura para compras agentes.

Esto hace que la preparación del catálogo sea una tarea central del AEO.

Lea la [guía de datos estructurados del producto](/es/docs/product-structured-data-ai-shopping/) relacionada para obtener detalles a nivel de esquema y la [preparación del carrito universal](/es/docs/universal-cart-agentic-commerce/) para conocer las implicaciones del carrito.

## Campos mínimos del catálogo

| Campo | Por qué los agentes lo necesitan |
|
---|
---|
| Identificación del producto | Coincidencia estable entre feed, página, carrito y pedido |
| Nombre | Etiqueta de entidad primaria |
| Categoría | Comparación y filtrado |
| Descripción | Comprensión de características |
| Atributos variantes | Tamaño, color, paquete, suscripción, región |
| Precio | Presupuesto y comparación |
| Disponibilidad | Previene carros fallidos |
| Política de envío | Costo total y elegibilidad |
| Política de devolución | Ajuste de riesgo y preferencia del usuario |
| Imágenes y texto alternativo | Contexto visual y accesibilidad |
| Reseñas/calificaciones | Señal de confianza y calidad |

## ¿Feed, esquema, API o UCP?

| Superficie | Mejor papel |
|
---|
---|
| HTML de la página del producto | Descubrimiento de humanos y rastreadores |
| Schema.org/Producto | Búsqueda e interpretación estructurada |
| Feed de comerciantes/productos | Distribución de plataformas |
| API de catálogo | Búsqueda en tiempo real de agentes |
| Capacidad de catálogo UCP | Interoperabilidad del comercio agente |

La mayoría de los comerciantes necesitarán más de una superficie. La parte importante es la coherencia.

## Fallos comunes del catálogo

- los precios difieren entre el feed y la página
- las variantes se fusionan en descripciones vagas
- los productos agotados todavía parecen disponibles
- los costos de envío aparecen solo al finalizar la compra
- las reglas de devolución están escritas como un texto legal vago
- las imágenes de los productos no tienen texto alternativo útil
- los paquetes no tienen componentes legibles por máquina
- las reglas de suscripción no están claras

Estos fracasos perjudican la conversión humana y la selección de agentes.

## Búsqueda de IA e implicaciones AEO

Para [optimización de búsqueda con IA](/es/docs/ai-search-optimization/), la preparación del catálogo mejora la calidad de las citas y las recomendaciones. Para [Optimización de Agent Engine](/es/docs/what-is-aeo/), admite el paso de la recomendación a la acción.Los agentes necesitan saber:

- cuál es el producto
- para quién es
- si se ajusta a las limitaciones del usuario
- si se puede comprar ahora
- qué sucede después de la compra

## Preguntas frecuentes

### ¿Es el esquema del producto suficiente para los agentes de compras con IA?

No. El esquema del producto ayuda, pero los agentes también pueden necesitar feeds, API, capacidades de carrito, datos de políticas e inventario actual.

### ¿Qué deberían solucionar primero los equipos de comercio electrónico?

Comience con ID de productos estables, claridad de variantes, precio y disponibilidad actuales, reglas de envío, política de devoluciones y un esquema de producto limpio.

### ¿Todos los productos deberían tener un punto final API?

No siempre. Pero los catálogos de alto valor, gran volumen o que cambian con frecuencia se benefician de las superficies de búsqueda en tiempo real.

### ¿Cómo afecta la preparación del catálogo a Universal Cart?

Los carritos universales o agentes se basan en datos precisos de artículos, precios, variantes y políticas. Los datos de catálogo débiles conducen a carritos fallidos o de baja confianza.
