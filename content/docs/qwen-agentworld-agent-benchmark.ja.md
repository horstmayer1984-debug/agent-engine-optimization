# Qwen AgentWorld と AEO

Qwen AgentWorld は、エージェント評価の変化を示唆しているため、AEO にとって重要です。モデルはテキスト タスクだけでなく、シミュレートされた環境でトレーニングされ、評価されます。 Web サイトでは、エージェントがアクション、状態遷移、結果を評価することを期待する必要があるため、ページとツールはテスト可能である必要があります。

## なんとハグ顔が出てきました

Hugging Face プラグインは、2026 年 6 月 28 日にトレンドのテキスト生成モデルとして [Qwen/Qwen-AgentWorld-35B-A3B](https://hf.co/Qwen/Qwen-AgentWorld-35B-A3B) として登場しました。モデル ページでは、エージェント、環境シミュレーション、ワールド モデル、および AgentWorldBench のラベルが付けられています。

関連記事 [Qwen-AgentWorld: 一般エージェントのための言語世界モデル] (https://hf.co/papers/2606.24597) は 2026 年 6 月 23 日に公開され、エージェント環境をシミュレートするための言語世界モデルについて説明しています。

Web サイト チームにとって、シグナルは単純です。エージェントは環境でテストされています。あなたのサイトもそうした環境の 1 つです。

## AEO にとってベンチマークが重要な理由

|参考コンセプト |ウェブサイトへの関与 |
|
---|
---|
|環境の状態 |ページには現在のステータスを明確に記載する必要があります。 |
|アクションシミュレーション |エージェントは、自分のアクションから予測可能な結果を​​必要とします。 |
|報酬 | Web サイトには測定可能な成功ステータスが必要です。 |
|長期的なタスク |複数ステップのワークフローは回復可能である必要があります。 |
|検証 |エージェントはタスクが完了したことを証明する必要があります。 |

これは、[エージェント評価パラメータ](/es/docs/agent-evaluation-benchmarks/)と[エージェント決定ツリー](/es/docs/agent-decision-trees/)に接続されます。

## Web サイトをベンチマークに適したものにする方法

1. タスクの開始状態と終了状態を定義します。
2. 危険なフローに対して安定したテスト アカウントまたはサンドボックス モードを提供します。
3. API 呼び出しの予想される結果を公開します。
4. 視覚的にのみ表示される、隠れた状態の変更は避けてください。
5. 予約、注文、発送には明示的な確認を使用します。
6. 状態遷移を記録します。
7. 同じタスクでエージェントを毎月テストします。

運用測定には、[AEO KPI](/es/docs/aeo-kpis-measurement/) を使用します。

## 推測してはいけないこと

Qwen AgentWorld は、すべての Web サイトがすべてのエージェントによって使用できることを実証するものではありません。これは、モデリング研究がどこに向かっているのかを示しています。エージェントには、アクションをシミュレートし、スコアを付け、改善できる環境が必要です。

|悪い推論 |最高のパフォーマンス |
|
---|
---|
| 「エージェントのベンチマークは Web サイトの UX を解決します。」 |これらにより、Web サイトのワークフローがどこで失敗したかがエージェントに公開されます。 |
| 「モデルタグは生産への可用性を保証します。」 |モデルのメタデータを研究および評価のコンテキストとして扱います。 |
| 「AEOは単なるコンテンツです。」 |エージェントの評価には、アクションと結果が含まれます。 |

##エージェントのベンチマークを Web サイトのテストに変える実際の Web サイトのウォークスルーから小さなタスクのセットを作成します。各タスクには、初期 URL または機能、許可されたツール、ユーザー制限、予想される終了状態、成功を判断するためのルールが必要です。少なくとも 1 つの無効なエントリと 1 つの中断されたワークフローを含めます。

発見、解釈、実行、検証に関してサイトを個別に評価します。モデルは正しいページを見つけても、ポリシーを読み間違える可能性があります。正しいアクションを選択することはできますが、結果を確認することはできません。単一の合格/不合格スコアでは、これらの違いが隠蔽されます。ベンチマーク結果は、サポートされていないモデルの評価を公開するためではなく、インターフェイスを改善するために使用してください。コンテンツ、スキーマ、またはエンドポイントの動作に対する後続の変更を公平に比較​​できるように、各結果でタスクのバージョンとサイトのバージョンを維持します。

## よくある質問

### Qwen AgentWorld は Web サイト最適化ツールですか?

いいえ、これはエージェント シミュレーションに関するモデルと研究の方向性です。 AEO 値の教訓は、Web サイトにはテスト可能な環境が必要であるということです。

### AgentWorldBench とは何ですか?

Hugging Face モデルのメタデータは、Qwen AgentWorld を、エージェント環境のシミュレーションに関連するベンチマークである AgentWorldBench にリンクします。

### サイトは独自のエージェント ベンチマークを作成する必要がありますか?

高価値のワークフローの場合は可能です。通常、問題を見つけるには、単純な繰り返しタスクのテストで十分です。

### これは電子商取引にどのような影響を与えますか?

製品の検索、カートの作成、ポリシーの確認、支払いの承認は、エージェントのタスクとしてテスト可能である必要があります。
