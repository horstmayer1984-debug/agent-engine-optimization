# 件のスモエージェント ツール呼び出し

スモラージェントは、コードベースのエージェントと構造化されたツール呼び出しエージェントの違いを簡単に確認できるため、AEO にとって重要です。明確なツール、回路図、安全な実行パスをレイアウトする Web サイトは、視覚的な推測に依存するサイトよりも、どのエージェント スタイルでも使いやすくなります。

## スモラジェントとは何ですか?

Hugging Face のドキュメントでは、最小限の抽象化でエージェントを作成するためのオープン ソース Python ライブラリとして [smolagents](https://huggingface.co/docs/smolagents/index) について説明されています。彼らのドキュメントでは、アクションをコードとして記述する「CodeAgent」と、JSON のような構造化されたツール呼び出しを使用する「ToolCallingAgent」の 2 つの主要なエージェント タイプに焦点を当てています。

GitHub プラグインは、[huggingface/agents-course](https://github.com/huggingface/agents-course) を含む Hugging Face エージェント エコシステムも明らかにしました。 Hugging Face プラグインには、スモラジェントとツールの周りにラベルが付けられた [First Agent Template Space] (https://hf.co/spaces/agents-course/First_agent_template) が表示されました。

## AEO の CodeAgent と ToolCallingAgent の比較

|エージェントの種類 |仕組み |ウェブサイトの要件 |
|
---|
---|
---|
|コードエージェント |ツールを呼び出すか、計算するためのコードを生成します。明確な例、安全なサンドボックス、決定的な結果 |
|ツールコールエージェント |ツールへの構造化された呼び出しを発行する |ツール名、JSON スキーマ、検証、エラー メッセージ |
|ブラウザエージェント |ビジュアルページの操作 |セマンティック UI、ラベル、安定したフォーム、明確な状態 |

AEO 戦略は、コード エージェント用のドキュメント、ツール呼び出し元用のスキーマ、ブラウザ エージェント用のアクセス可能なユーザー インターフェイスの 3 つを関連するすべてでサポートする必要があります。

## ツールを呼び出すエージェントに対して公開するもの

1. アクションを説明するツール名。
2. 必須フィールドを含むスキーマを入力します。
3. 成功状態とエラー状態を含むスキームを出力します。
4. レート制限と認証ルール。
5. 人間による承認ルール。
6. 本番環境の動作に一致する例。
7. ツールが変更された場合のリリースノート。

プロトコル レベルのガイダンスについては、[MCP リソース対ツール対プロンプト](/es/docs/mcp-resources-tools-prompts/) および [MCP 対エージェント API](/es/docs/mcp-vs-api-for-agents/) を参照してください。

## ウェブサイトへの影響

smolagents はウェブサイト最適化ツールではありません。これはエージェント作成者の考え方を表しています。彼らは曖昧なページを望んでいません。具体的な行動が必要です。

|ページの種類 |エージェントリストの改善 |
|
---|
---|
| API ドキュメント |操作例、エラー、認証状態を追加します。 |
|価格ページ |比較表でプランの制限を公開します。 |
|電子商取引ページ |バリエーション、在庫、価格、配送、返品を表示します。 |
|サポートページ |インシデントをタイプと必要なデータ別にルーティングします。 |

実際の目標は [実行層](/es/docs/execution-layer/) です。エージェントは、ページの読み取りから、検証と回復を伴う限定されたタスクを完了するまでの作業を行う必要があります。

## よくある質問

### スモラジェントは開発者専用ですか?このライブラリは開発者向けですが、エージェントはより優れた Web サイト インターフェイスを必要としているため、その影響はマーケティング担当者、製品チーム、ドキュメントの所有者に影響します。

### Web サイトは Python コードをエージェントに公開する必要がありますか?

一般的にはそうではありません。公開 Web サイトでは、安定したデータ、API、ドキュメント、ツールを公開する必要があります。コードの実行は制御された環境に属します。

### コード エージェントと JSON ツール呼び出しのどちらがより安全ですか?通常、構造化されたツール呼び出しは検証が簡単です。コード エージェントはより柔軟にすることができますが、堅牢なテスト環境が必要です。

### これは AEO とどのように関係しますか?

AEO を使用すると、エージェントはコンテンツとアクションを簡単に発見、解釈、実行、検証できます。
