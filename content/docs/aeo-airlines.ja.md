航空会社は、価格と時間が最も重要な旅行市場の 1 つで事業を展開しています。動的価格は分ごとに変化します。お席の在庫は常に変動しております。付帯サービス (荷物、座席、食事) により、それぞれの比較がさらに複雑になります。 AI エージェントは、数百ものレートの組み合わせを数秒で処理できるため、まさにこの環境で成功します。これは人間が手動で実行することはできません。

1 秒未満の応答時間で構造化されたフライト データを提供する航空会社は、代理店による予約の割合が増加しています。低速なシステム、隠れたアドオン価格設定、または視覚的なインタラクションを必要とする予約フローを備えた航空会社は、より高速な競合他社に負けます。

## 航空システム担当者が必要とするもの

### リアルタイムの在庫状況データと料金

動的価格設定とは、エージェントに表示される価格が予約時に適用される価格である必要があることを意味します。たとえ数ユーロであっても、相違があると信頼の喪失を引き起こします。エージェントは矛盾を記録し、将来の参照のために航空会社を格下げします。

運賃 API は、出発地、目的地、日付 (柔軟および固定)、乗客の数と種類 (大人、子供、乳児)、キャビン クラスの希望およびオプションの制限 (直行便のみ、特定のアライアンス、最大乗り継ぎ時間) を受け入れる必要があります。

応答には、乗客ごとの正確な運賃、運賃クラス、税金と手数料の内訳、予約クラスの空席状況、および運賃の有効期間 (この価格が保証される期間) が含まれている必要があります。

### 補助サービスの体系化された価格

手荷物、座席の選択、食事、ラウンジへのアクセス、優先搭乗、旅行保険の価格は、別途体系化された項目として見積もられる必要があります。旅行の総費用を計算するエージェントは、オプションが表示される「予約の管理」リンクではなく、各参加者の正確な数値を必要とします。

付随サービスの価格を、各料金クラスにリンクされた構造化データとして公開します。 23kgの受託手荷物を含む運賃には、これを明示する必要があります。別途請求される料金には、手荷物ごとの正確な料金が含まれている必要があります。

### 決定的なチケット発行エンドポイント

発券エンドポイントは、乗客の詳細、選択されたフライト、選択された付随サービス、運賃クラス、および支払い承認を受け入れます。戻り値: PNR (予約参照) 付きの確認済みチケット、電子チケット番号、正確な合計金額、および特定のレートを含むキャンセル/変更条件。

応答は即時に行われなければなりません。 「処理中です。後で確認してください」を返すチケット発行エンドポイントは、エージェントにポーリング ロジックを実装し、不確実な状態を処理することを強制します。即時確認、または特定の理由による即時拒否は、標準エージェントが期待するものです。

### 変更・キャンセル条件を構造化データとして表現

各運賃クラスは、構造化された変更およびキャンセル条件、つまり運賃額、期限（出発の数時間前）、払い戻し方法（当初の支払い、バウチャー、なし）、および名前変更ポリシーを公表する必要があります。

ビジネス旅行者向けのフライトを評価する代理店は、キャンセルの柔軟性を高く評価しています。これらの規約を構造化データとして公開している航空会社は、代理店が法的文書を分析する必要がある航空会社よりも優先されます。

##動的価格設定の利点航空会社はすでに高度な収益管理システムを運用しています。 AEO の機会は、双方に利益をもたらす方法でこれらのシステムをエージェントに公開することです。

リアルタイムで運賃を確認し、航空会社を比較し、即座に予約できるエージェントは、航空会社の履行率を高めます。この航空会社は、従来のチャネルではおそらく発生しなかったであろう販売を達成しました。エージェントはユーザーに対して最良のレートを取得します。

重要な要件: API 経由で見積もられた価格は、チケット発行時の価格と一致する必要があります。出品と購入の間で価格が変動するため、取引の失敗が生じ、エージェントの信頼が永久に損なわれます。

## よくある質問

**航空会社は代理店向けの動的価格設定をどのように処理していますか?**
リアルタイム レート API を介して、有効期間 (通常は 15 ～ 30 分) で現在の価格を返します。エージェントはその期間内に予約するか、現在の料金を再度確認する必要があります。

**エージェントは旅行代理店や OTA に代わるものですか?**
単純なポイントツーポイント予約の場合、その可能性はますます高まっています。複雑な複数区間の旅程の場合、エージェントは人間の旅行代理店と協力して作業します。堅牢なエージェント向け API を提供する OTA は、引き続きアグリゲーション レイヤーとして機能します。

**NDC (新しい配電容量) についてはどうですか?**
NDC は、より充実した構造化された料金配分を目的として設計されているため、AEO とよく連携します。すでに NDC を導入している航空会社は、エージェント対応の API インフラストラクチャにおいて有利です。

**補助サービスの価格の透明性はどの程度重要ですか?**非常に重要です。本当の総コスト (運賃、手荷物、座席) を計算できないエージェントは、自分の競争力を過大評価または過小評価することになります。どちらも改宗に悪影響を及ぼします。

**レート API の目標応答時間はどれくらいですか?**
可用性クエリの場合は 500 ミリ秒未満。チケットの発券には 3 秒もかかりません。代理店は数十の航空会社を同時に比較し、反応が遅い航空会社にペナルティを課します。
