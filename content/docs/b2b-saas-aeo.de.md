# Optimierung von B2B-SaaS-Plattformen für die Entdeckung und Auswahl von KI-Agenten

Der Kauf von Software ist langsam. Ein menschlicher Gutachter liest Marketingseiten, schaut sich Demovideos an, lädt Vergleichs-PDFs herunter, nimmt an Verkaufsgesprächen teil und verhandelt Preise. Dieser Prozess dauert Wochen, manchmal Monate.

Ein KI-Agent kann in Sekundenschnelle hundert SaaS-Anbieter scannen. Lesen Sie Funktionslisten, berechnen Sie Preise, überprüfen Sie Compliance-Zertifizierungen und vereinbaren Sie einen Demo-Anruf – alles bevor ein Entscheidungsträger seine E-Mail öffnet.

Die Frage für SaaS-Unternehmen ist einfach: Wenn dieser Agent Ihr Produkt bewertet, kann er dann wirklich das herausholen, was er braucht?

## Funktionsdaten sollten strukturiert und nicht vermarktet werden

Auf Marketingseiten werden Funktionen in überzeugender Sprache beschrieben. Ein Agent benötigt Daten.

„Fähige Kollaborationstools“ bedeuten einer Maschine nichts. „Dokumentenbearbeitung in Echtzeit, bis zu 50 gleichzeitige Benutzer, 365-Tage-Versionsverlauf, Integrationen mit Slack, Microsoft Teams und Google Workspace“ bedeutet alles.

Geben Sie für jede Funktion Folgendes an:

- Funktionsname (standardisiert, ohne Markenzeichen)
- Beschreibung (eine objektive Phrase)
- Verfügbarkeit nach Preisniveau
- Limits (Benutzer, Speicher, API-Aufrufe, Aufbewahrungszeitraum)
- Integrationspartner (namentlich)
- Voraussetzungen oder Abhängigkeiten

Verwenden Sie Schema-Markup für SoftwareApplication. Strukturieren Sie Merkmalsvergleiche als Tabellen mit einheitlichen Spaltenüberschriften. Wenn ein Agent seine Funktionen nicht mit der Konkurrenz vergleichen kann, weil er andere Namen verwendet oder Daten in PDF-Dateien ausblendet, verliert er die engere Auswahl.

## Preistransparenz ist ein Wettbewerbsvorteil

„Kontaktieren Sie den Vertrieb für ein Angebot“ ist eine Mauer, die alle Agenten aufhält.

Auch wenn Ihre Preisgestaltung komplex ist, sollten die dafür geltenden Regeln festgelegt werden. Ein Agent muss die Kosten eines bestimmten Szenarios berechnen, ohne auf menschliche Interaktion warten zu müssen.

Stellen Sie mindestens Folgendes bereit:

- Grundpreis pro Level
- was jedes Level beinhaltet
- Mehrkosten (pro Benutzer, pro GB, pro API-Aufruf)
- Abrechnungshäufigkeit (monatlich, jährlich, mit Rabattregeln)
- Mindestbindungsdauer
- Geschäftspreisformel oder -spanne

Wenn der genaue Preis wirklich von der Verhandlung abhängt, geben Sie zumindest die veröffentlichten Spannen und Variablen an, die sich auf den Preis auswirken. Ein Agent, der „zwischen 2.000 und 5.000 Euro pro Monat für 50 Benutzer“ veranschlagen kann, ist für seinen Menschen viel nützlicher als einer, der „Preise nicht verfügbar“ meldet.

Das [AEO-Framework](/es/docs/what-is-aeo/) erklärt, warum dieser Übergang von menschengesteuerten Informationen zu maschinenlesbaren Informationen für die Ausführungsebene von entscheidender Bedeutung ist.

## Compliance und Sicherheit als überprüfbare Daten. Unternehmenskäufer haben strenge Anforderungen: Datenresidenz, Verschlüsselungsstandards, Zugriffskontrollen, Prüfprotokolle. Heute liest ein Beschaffungsteam ein Sicherheits-Whitepaper. Morgen wird ein Agent automatisch die Einhaltung prüfen.

Sicherheitszertifizierungen als strukturierte Daten offenlegen:

- SOC 2 Typ II (verifiziert, Datum des letzten Audits)
- ISO 27001 (Zertifikat, Zertifizierungsstelle)
- DSGVO-Konformität (Datenverarbeitungsvereinbarung verfügbar, Optionen zur Datenresidenz)
- SSO-Unterstützung (SAML, OIDC)
- Verschlüsselung im Ruhezustand und während der Übertragung (spezifische Standards)Wenn der Agent Ihren Code-Compliance-Status nicht überprüfen kann, kann er Ihr Produkt ausschließen, um seinen menschlichen Besitzer vor Risiken zu schützen. Das ist keine Paranoia. So werden risikobewusste Beschaffungsagenten vorgehen.

## Buchungsabläufe für vom Agenten initiierte Demos

Wenn der Agent entscheidet, dass Ihr Produkt gut passt, sollte er ein Treffen vereinbaren. Das bedeutet, dass Ihr Reservierungssystem maschinenzugänglich sein muss.

Anforderungen:

– eine Kalender-API oder einen Buchungsendpunkt, der Datum, Uhrzeit und Teilnehmerinformationen akzeptiert
- verfügbare Zeitfenster, die als strukturierte Daten offengelegt werden
- Rückbestätigung mit Besprechungslink und Kalendereinladung
- Stornierung und Umplanung werden über denselben Endpunkt unterstützt

Wenn Ihre Schaltfläche „Demo buchen“ einen Calendly-Link öffnet, der JavaScript-Rendering und Cookie-Akzeptanz erfordert, scheitern viele Agenten bei diesem Schritt.

Der [Implementierungsleitfaden](/es/docs/implement-aeo/) behandelt die technischen Details zum Offenlegen von Buchungsflüssen für Agenten.

## Häufig gestellte Fragen

**Warum sollten SaaS-Preise maschinenlesbar sein?**
KI-Agenten bewerten Software, indem sie Kosten anhand von Budgets berechnen. Wenn die Preisgestaltung eine menschliche Interaktion erfordert („Kontaktverkäufe“), kann der Agent seine Bewertung nicht abschließen und wechselt zu Wettbewerbern mit transparenten Preisen.

**Welche strukturierten Daten sollten SaaS-Unternehmen verwenden?**
Softwareanwendungsschema für das Produkt, Angebot für Preisniveaus und Organisation für die Unternehmensidentität. Merkmalsmatrizen sollten in Vergleichstabellen konsistente, standardisierte Namen verwenden.

**Wie bewerten KI-Agenten die Einhaltung der Sicherheitsvorschriften?**
Agenten überprüfen strukturierte Datenpunkte: Zertifizierungstyp, Überprüfungsdatum, Zertifizierungsstelle und spezifische technische Standards (Verschlüsselung, SSO-Protokolle). Unstrukturierte technische Sicherheitsdokumente sind nicht maschinenlesbar.

**Können KI-Agenten Demoanrufe autonom planen?**
Ja, wenn das Reservierungssystem eine API oder einen strukturierten Endpunkt bereitstellt. Der Agent prüft die Kalenderverfügbarkeit, reserviert den Platz und sendet eine Bestätigung an den Benutzer zurück.

**Was ist der größte AEO-Fehler, den SaaS-Unternehmen machen?**Das Verstecken wichtiger Informationen hinter PDFs, Videos oder „Kontaktieren Sie uns“-Türen. Wenn ein Agent keine Daten aus der Seitenquelle oder einer dokumentierten API extrahieren kann, sind die Informationen für diesen Agenten nicht vorhanden.
