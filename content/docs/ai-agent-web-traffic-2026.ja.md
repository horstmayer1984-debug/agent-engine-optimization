# 2026 年の AI エージェントの Web トラフィック

AI エージェントの Web トラフィックは急速に増加していますが、最も堅牢なレポートは異なるものを測定します。 BrightEdge によると、AI エージェントのリクエストは人間のオーガニック検索活動の 88% に達しました。ボット セキュリティ レポートによると、自動化されたトラフィックが Web 運用における大きな問題となっています。ウェブサイト所有者は、ページビューだけでなくエージェントの検索を追跡する必要があります。

## 数字がわかりにくく聞こえる理由

すべてのヘッドラインの市場数値が同じ分母を測定するわけではありません。

BrightEdge は 2026 年 4 月に、[AI エージェントのリクエストが人間のオーガニック検索アクティビティの 88% に達した](https://www.brightedge.com/news/press-releases/brightedge-data-ai-search-reaching-tipping-point-ai-agents-2026) と報告しました。これは検索に関連したベンチマークであり、AI エージェントがインターネット トラフィック全体の 88% を占めているという主張ではありません。

Thales と Imperva の [2026 Bad Robots Report](https://www.imperva.com/resources/resource-library/reports/2026-bad-bot-report/) は、問題を別の角度から捉えています。現在、ロボットが世界の Web トラフィックの大部分を占めており、エージェント AI により自動化された意図の検出が困難になっています。

HUMAN の [2026 年の AI トラフィックおよびサイバー脅威ベンチマーク レポートの現状] (https://www.humansecurity.com/learn/resources/2026-state-of-ai-traffic-cyberthreat-benchmarks/) も、AI トラフィックを単なるマーケティングの問題ではなく、セキュリティと信頼管理の問題として扱っています。

有用な結論は単純です。AI エージェントとシステムは実際のアクセス パターンになりつつあります。あなたの分析スタックはおそらくそれらを過小評価しています。

## AI エージェントのトラフィックと通常のボットのトラフィック

|トラフィックの種類 |典型的な意図 |ウェブサイトの対応 |
|
---|
---|
---|
|検索トラッカー |検索結果のページのインデックス |重要なページを許可し、クロール予算を管理する |
| AIトラッカー | AI システムのコンテンツを収集または取得する |何を許可するかを決定し、ボリュームを監視し、有用なコンテンツへのアクセスを維持します |
|ブラウザエージェント |ユーザーに代わって行動する |フォーム、アクション、確認を明確にする |
|スクレーパーロボット |ユーザー関係なしでデータを抽出する |ポリシーに基づいたレートキャップまたはブロック |
|詐欺ロボット |アカウント、支払い、広告、API を攻撃する |ブロック、異議申し立て、またはより強力な検証を要求する |

難しいのは、ユーザーの利益と悪用が HTTP 層では同様に見える可能性があることです。正規のショッピング エージェントも有害なスクレーパーも、マシンの速度で商品ページをリクエストできます。

## Web サイト所有者が追跡すべき内容

小規模な測定セットアップから始めます。

1. `/llms.txt`、マークダウン ページ、API 仕様、製品フィード、およびドキュメントのリクエスト。
2. ChatGPT、Perplexity、Claude、Gemini、Copilot、およびその他の AI サーフェスからの参照。
3. 既知の AI ボットおよびクローラー ユーザー エージェント。ただし、リストは不完全であると想定されます。
4. リクエストが通常のマルチページ セッションを置き換えるサーバー ロギング パターン。
5. AI支援セッション後のコンバージョンルート。
6. 自動化された顧客が使用するフォームおよび支払い手順のエラー率。7. AI トラッカーとブラウザ エージェントからのリクエストをブロックしました。

SEO側の計測には【AI Referral Attribution for AEO SEO】(/es/docs/ai-referral-attribution-seo/)をご利用ください。アクセス制御ポリシーについては、[AI クローラーと robots.txt](/es/docs/ai-crawlers-robots-txt/) および [AI クローラーの Web ボット認証](/es/docs/web-bot-auth-ai-crawlers/) を参照してください。

## アクセス制御の決定|決定 | | の場合に使用します。リスク |
|
---|
---|
---|
|許可 |エージェントは、ユーザーがサイトを発見または使用できるように支援します。インフラストラクチャ負荷の増加 |
|レート制限 |交通量は便利ですが、頻度が多すぎます。一部のエージェントはタスクに失敗する可能性があります。
|認証を要求する |このタスクには、アカウント、価格、個人データ、またはアクションが含まれます。さらなる統合作業 |
|ブロック |トラフィックはポリシーに違反し、損害を与え、またはビジネス ルールを回避します。 AI ワークフローで失われた可視性 |

すべての自動トラフィックをブロックすることは、中立的なデフォルトのオプションではなくなりました。リソースを保護することはできますが、エージェントによるツアーからサイトを削除することもできます。

##実践的なエージェント トラフィック チェックリスト

1. どの AI エージェントとトラッカーが許可されるかを文書化します。
2. クローラーに対して「robots.txt」を明示的に保ちます。
3. ページビューだけでなく、アクションのセキュリティ制御を追加します。
4. AI 関連のパスがないかサーバー ログを毎週監視します。
5. エージェント向けの AI 紹介とファイル リクエスト用のダッシュボードを作成します。
6. ブラウザ エージェントが推測せずに安全なタスクを完了できるかどうかをテストします。
7. 優れた自動化と政治的言語による乱用を区別する。

これは、[エージェント UX と AEO SEO](/es/docs/agent-ux-aeo-seo/) および [Lighthouse Agentic Browsing Audit](/es/docs/lighthouse-agentic-browsing-audit/) に接続します。

## よくある質問

### AI エージェントは本当に今の Web トラフィックのほとんどを占めているのでしょうか?

一部のレポートでは、現在、自動トラフィックが世界の Web トラフィックの大部分を占めており、BrightEdge によると、AI エージェントのリクエストは人間のオーガニック検索アクティビティの 88% に達しています。これらを 1 つの請求項にまとめないでください。彼らはさまざまなことを測定します。

### なぜ分析は AI エージェントからの訪問をスキップするのでしょうか?

エージェントのフェッチの多くは、JavaScript の解析、参照の保持、スクロール、クリック、または通常のページ パスの実行を実行しません。サーバーログが必要です。

### Web サイトは AI エージェントをブロックする必要がありますか?

デフォルトではありません。タスク、価値、リスク、リソースコストによって決定します。クローラー アクセス、ブラウザー エージェントのアクション、および認証されたワークフローには、異なるルールが必要です。

### 最初に追加するメトリクスは何ですか?

`/llms.txt` リクエスト、API 仕様、マークダウン コンテンツ、既知の AI 参照、エージェントの使用を示唆するサーバー ログ パターンを追跡します。
